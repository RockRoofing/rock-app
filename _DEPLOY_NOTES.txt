DEPLOY NOTES - Sales: bar = table = Xero (single source, + bank receipts)
========================================================================

THE CAUSE OF THE MISMATCH
-------------------------
The BAR came from the P&L benchmark; the TABLE summed the lines I pulled. They diverged
month-to-month for two reasons:
  1. The table was MISSING bank receipts (receive money) coded to sales - the P&L
     includes those, so the table came out low on months with bank-coded sales.
  2. Bar (P&L snapshot) and table (live lines) could be from slightly different points
     in time, so each matched Xero in different months.

THE FIX (makes all three agree by construction)
-----------------------------------------------
1) Added BANK TRANSACTIONS coded to sales (receive money = +, spend money = -) to the
   pulled lines. The table now contains every transaction type that hits code 200:
   invoices, WIP manual journals, credit notes AND bank receipts - the same set the P&L
   sums.
2) The BAR is now built by summing THOSE SAME LINES per month (not from the P&L
   benchmark). So the bar and the table total are now ALWAYS identical.
3) Because the lines now match what the P&L is made of, both also match Xero.

CROSS-CHECK BUILT IN
--------------------
Under the chart there's now a P&L cross-check for the visible period:
  "P&L cross-check: app <x> vs P&L <y> (match)" - or it shows the difference.
If every month is captured, it will say (match). If a month still differs, that tells us
a sales entry is posted in yet another way - send me that month and I'll add the source.
