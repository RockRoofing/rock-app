Replaces the batch 3a pages/retention.js (deploy over the top).

WHAT CHANGED
------------
When a project's VAT Type is "Mixed" (its invoices carry different VAT
treatments), VAT can't be worked out from Final Account × a single rate. So:

• The VAT column now shows an amber "⚠ enter manually" flag (with a hover
  explaining why) instead of a wrong auto-figure.
• A new "Manual VAT £" field in the Edit form lets you type the correct VAT.
  It's marked "(required — mixed)" when the type is Mixed. A manual VAT figure
  always takes precedence (also works as an override for any row).
• Total Due / Total Remaining use the manual VAT once entered, so they compute
  correctly.
• A Mixed project will NOT show as closed (row stays non-green) until its VAT has
  been entered manually — so nothing is falsely marked settled.

FILE CHANGED
------------
• pages/retention.js — Mixed-VAT flag in the VAT column + Manual VAT field;
  closed check waits for manual VAT.
