RAMS — Resend dropdown now populated + Download RAMS as a button
===============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of the previous RAMS fixes.

1) "RESEND RAMS TO SITE MANAGER" — dropdown was empty
----------------------------------------------------
Cause: the dropdown only listed projects whose current RAMS was EXACTLY at the
Site Manager stage. A project where the Site Manager had already approved (now at
operatives) — or any project past that point — showed nothing, so the list came
up empty.
FIX: a project is now eligible to resend once the DIRECTOR has signed, i.e. its
current RAMS is at the Site Manager stage OR beyond (site-manager / operatives /
complete). Sending re-issues a fresh approval link, RE-OPENS the Site Manager
stage (so their new approval takes effect), invalidates the old link, and emails
the Site Manager again.
The server's set-site-manager action was widened to match (allowed once the
Director has signed; blocked before that and on rejected RAMS).

2) "DOWNLOAD RAMS" IS NOW A BUTTON (Site Manager approval page)
--------------------------------------------------------------
On the customer Site Manager page, the download is now a solid gold "⬇ Download
RAMS" button aligned to the right under the preview (previously a centered text
link).
