DEPLOY NOTES - VAT Refund: filed returns (past) + estimate (current/future)

THE MODEL (agreed)
------------------
- COMPLETED months  -> Xero's FILED VAT return (the real figure - it already handles late
  claims, domestic reverse charge, zero-rating and the retention accounts, which a simple
  transaction sum never can).
- CURRENT + FUTURE months -> a transaction ESTIMATE, because no return exists yet. Clearly
  labelled "Estimate". It's indicative for cash flow and moves as invoices/bills post.

The table now has a Source column: green "Filed return" vs amber "Estimate".

WHY THE OLD VERSION WAS WRONG
-----------------------------
Your VAT return xlsx showed the return includes: late claims from prior periods (back to
Oct 2024), DRC reverse charge on CIS labour (adds to BOTH box 1 and box 4), zero-rated
income, and retention (612) lines. Summing TotalTax by month can't reproduce that - hence
the monthly mismatches. So past months now come straight from the filed return.
