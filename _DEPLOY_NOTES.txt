2) ANTICIPATED FINAL ACCOUNT — SENT APPLICATION IS THE SOURCE OF TRUTH
   When an application is marked sent (via "Mark as sent") OR sent from the platform
   (the send-to-customer flow with "mark as sent"), that application's Anticipated
   Final Account (measured contract sum + final variations) is stored on the project
   as afaOverride and OVERWRITES the AFA shown in:
     • Project Financials (project detail page)
     • Budget Tracker (commercial dashboard)
     • Retention register
   Until an application has been sent, AFA falls back to the computed
   contract value + instructed variations (unchanged behaviour).
   The dashboard cache is cleared on send, so the three surfaces update immediately.

   NOTE: the latest sent application wins (it re-writes afaOverride each send), which
   is the intended "application data is the source of truth" behaviour.
