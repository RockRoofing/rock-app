SITE APP + PORTAL — PDF viewer (canvas) + task fixes

DRAWINGS / RAMS + DELIVERIES PDF VIEWING
  The reason PDFs kept showing the browser's "Open / download" prompt is that mobile Chrome refuses to
  inline-render a PDF in an <iframe>, no matter the headers. Fix: PDFs are now rendered with pdf.js onto
  <canvas> elements (scrollable, all pages), which the browser cannot hijack.
  - pages/forms/index.js: FileViewer now uses a PdfCanvas component for PDFs (Drawings & RAMS). Images
    still render via the inline proxy.
  - components/RowAttachments.js: the shared AttachmentViewer (Deliveries/Procurement/Risk/Tasks) now uses
    the same pdf.js canvas for PDFs.
  pdf.js is loaded from the same CDN already used for thumbnails; PDFs load via /api/download?inline=1 so
  pdf.js fetches them same-origin.

PORTAL LIVE PROJECT TASKS (pages/operations/project-management/tasks.js)
  - Target Completion date is now editable INLINE (date picker in the cell), like the other schedules.
  - When no target date is set, the cell shows "Target Completion Date Required" (was "No target date").

Deploy all three files. No cron/env change.
