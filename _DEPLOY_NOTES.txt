WHAT THIS ADDS
--------------
1) P&L BENCHMARK NOW COVERS EVERY EXPENSE CODE (incl. OVERHEADS)
   Previously the Xero P&L benchmark only captured Income + Cost of Sales per code.
   It now ALSO captures the Overheads / Expenses sections, tagged by section
   (income / cos / overheads). So the reconciliation can compare every categorised
   code — Materials, Labour AND Overheads — against Xero, over the same months.
   • lib/xero.js fetchProfitAndLoss: byCode + codeSection + overheadsTotal.
   • sync-benchmark stores codeSection + overheadsTotal.
   ⚠ You must click "Sync Xero figures" after deploy to repopulate the benchmark
     with the new per-code overhead data.

2) MISSING-LINES / ACCRUALS DETECTOR  (the gap you asked to close)
   The Bookkeeping app now compares, per account code, Xero's P&L amount vs the app's
   own bill+wage LINE data for the same code over the synced months. Any code where
   Xero shows MORE than the app can see (> £1) is surfaced in a collapsible panel:
   Code · Account · Category · Xero P&L · App lines · Difference.
   This is exactly the accruals / manual-journal gap — costs that hit Xero's P&L but
   never came through as an imported bill or wage line.

HOW TO BALANCE TO XERO NOW
--------------------------
1. Sync bills, wages and invoices (existing buttons).
2. Click "Sync Xero figures" (benchmark) — now with full per-code P&L incl overheads.
3. Categorise any Uncategorised codes (banner).
4. Work the "more in Xero than the app can see" panel to zero — each remaining line
   is a journal/accrual to check in Xero. When that panel is empty and uncategorised
   is zero, the app ties to the Xero P&L per code.
