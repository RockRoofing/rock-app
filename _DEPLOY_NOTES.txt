FORMS — revised due-logic (notifications + Missing dashboard kept in sync)

How items clear on the Missing dashboard:
  An item flips Missing -> Completed when a matching Site App submission arrives for that job/project within
  that form's week (site diaries match the specific day; others match the week). It's driven by real
  submissions, not manual ticking.

Rule changes (applied to BOTH the Monday cron and the Missing dashboard so they agree):

  PRE-START (CM)
   - Now triggers when a project STARTS or RETURNS to site within 14 CALENDAR DAYS of the Monday (was
     "within 3 working days"). Email is titled "Up and coming projects that require a Pre-Start
     Notification".

  DAILY SITE DIARY (Supervisor)
   - Email now lists the PROJECTS that need a site diary that week (was a count). Dashboard still tracks one
     per allocated day. Goes to the effective supervisor (Gantt-allocated supervisor takes priority over the
     Project Details one; if two, the first is used).

  WORKS AREA HANDOVER (Supervisor)
   - Now triggers when a finish day is followed by a gap of 5+ CALENDAR DAYS before the next visit (covers
     "finishes Friday, not back by the following Wednesday"), and also at project end (no next visit). Was
     "week of the last allocated day".

  START ON SITE CHECKLIST (Supervisor) — unchanged (week of the first allocated day).

  One combined email per supervisor still covers all their required forms (Start on Site + Site Diaries +
  WAH). CMs get their own Pre-Start email.

Deploy both over previous. No cron/env change.
