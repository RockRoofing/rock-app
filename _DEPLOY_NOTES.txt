WHY VAT TYPE WAS BLANK AND PAID WAS 0
-------------------------------------
Two caching issues:
1) The dashboard serves a 4-hour cached snapshot of all projects. If that snapshot
   was built before the new fields existed, it had no paid/vat/vatRateLabel — so
   the tracker showed 0 / blank regardless of a Xero sync.
2) A manual override row (created when you tick "Received") was freezing a copy of
   the row WITHOUT the live Xero financials.

WHAT THIS FIXES
---------------
• The dashboard now DERIVES paid / VAT / VAT-type / invoiced from the per-invoice
  lines cache as a fallback when the aggregate is missing them. "Paid" works
  immediately (AmountPaid has always been cached per invoice). VAT + VAT Type
  populate from the lines once a sync has stored per-line tax (batch 2b onward).
• The Retention Tracker now LAYERS the live Xero financials (invoiced/VAT/VAT
  Type/paid) on top of any manual override, so ticking "Received" no longer wipes
  those figures — they always reflect the latest Xero data.
