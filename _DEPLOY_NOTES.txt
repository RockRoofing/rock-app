ATTACHMENTS — in-app viewer + real download (Deliveries, Procurement, Risk Log, Live Tasks)

These four portal pages all share components/RowAttachments.js, so fixing it fixes all of them.

  components/RowAttachments.js
   - VIEW now opens an IN-APP viewer overlay (like the Drawings viewer) instead of opening the raw URL in
     a new tab: images render inline, PDFs embed, other types offer a download. Includes prev/next via
     on-screen ‹ / › arrows, "‹ Previous / Next ›" buttons, left/right arrow keys, and swipe. Close with ×
     or Escape — you never leave the page.
   - DOWNLOAD now saves the actual file with its correct name and extension (e.g. .jpg/.pdf) instead of a
     generic "file". It routes through a new same-origin download proxy that sets the filename; falls back
     to a blob fetch, then to opening the URL.
   - Small thumbnails now show next to image attachments in the list. Optional readOnly prop hides the
     add/remove controls (not required by the existing callers).

  pages/api/download.js  (NEW)
   - GET /api/download?url=<blob url>&name=<filename> — streams a Vercel Blob file back through our origin
     with Content-Disposition: attachment so the browser saves it correctly. Restricted to
     *.blob.vercel-storage.com hosts.

  middleware.js
   - Added /api/download to the Site App passthrough (so attachments can also be downloaded from Site App
     contexts).

No change needed to deliveries.js / procurement.js / risks.js / tasks.js — they already import
RowAttachments. Deploy all three files. No cron/env change.
