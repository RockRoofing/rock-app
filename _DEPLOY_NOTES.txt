CRITICAL FIX — PER-DAY REPLACE (was wiping older data)
------------------------------------------------------
The previous replace-by-range used the file's min..max span. A single stray-dated
line in a file (e.g. a corrected 2024 bill inside a "June 2026" export) made the
span 2024→2026 and WIPED everything in between across all projects. THIS is why
your older bills/wages vanished.

Now it replaces ONLY the exact DAYS present in the file:
  • Build a set of the distinct dates in the file.
  • Clear existing lines only on those exact days (across projects + untagged).
  • A stray date can only ever affect its own single day — never a span.
Runtime-tested: a June file containing one stray 2024-03-10 line updates only that
one day; 2025-11 and everything else survive.

DATA MANAGEMENT PANEL (Admin -> Data Management)
------------------------------------------------
New admin tab with wipe buttons for a clean rebuild:
  • Bills (Costs) | Direct Wages | Sales Invoices | Overheads — each with a
    "all X data will be permanently deleted" warning.
  • "Clear ALL financial data" for a full reset.
  • Each requires typing CLEAR to confirm. Admin-only. Does not touch projects,
    forms, users or non-financial data.
Recommended clean-rebuild flow: clear the type -> re-upload the Xero exports.

"DAYS NOT COVERED" WARNING (on bill uploads)
--------------------------------------------
After a bills upload, if the app has bills on days the file DIDN'T cover, it lists
them (date + value) with an explanation: the upload only changed the days it
contained; if any of those other days were edited/deleted in Xero, upload a file
covering them or check Xero. Turns the one residual per-day edge case into a
visible, checkable list.

RECOMMENDED NEXT STEPS (your clean rebuild)
-------------------------------------------
1. Deploy this + part 1.
2. Admin -> Data Management -> Clear Bills (type CLEAR). Repeat for Wages if
   rebuilding those too.
3. Re-upload your bill exports period-by-period (any order). Each replaces only

4. Wages: re-upload the correctly project-tagged exports (Projects column).
