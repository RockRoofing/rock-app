Wages sync — always dated + no duplicates (supersedes Wages_sync_dedupe_fix)
===========================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Single file: pages/api/sync-wages.js  (deploy BEFORE re-syncing wages)

DECISION
--------
Wages stay in the COSTS tab (they're a cost — a separate tab would break the
"all project spend in one place" reconciliation). For an accurate EOM report every
wage line MUST carry a date, so the fix makes undated wages impossible rather than
trying to display them.

WHAT THIS DOES
--------------
1) EVERY wage line is dated. The date now resolves from the Xero journal's Date,
   then DateString, then UpdatedDateUTC. If none resolve, the line is SKIPPED (and
   counted as skippedNoDate in the sync response) — never stored undated. So wages
   will always fall into the correct month in the Costs tab and the EOM report.

2) NO MORE DUPLICATES. The old sync kept every existing line that was before the
   window OR had no date, so undated lines were re-kept on every sync while fresh
   dated wages were added alongside — the doubling you saw on Russell Hill. The
   bucket is now rebuilt each sync: older DATED history + fresh in-window lines,
   de-duplicated by Xero JournalLineID (content-key fallback), legacy undated/no-id
   orphans dropped. Applied to per-project and untagged buckets.

AFTER DEPLOY
------------
1. Deploy this file.
2. Re-run the Wages sync (Bookkeeping page or commercial SyncBar) to repopulate the
   wages you cleared. They come in clean and fully dated.
3. Check the sync result — if "skippedNoDate" is > 0, those journals in Xero have no
   usable date and need a date set in Xero, then re-sync. (Expected to be 0.)
4. Wages now appear in each project's Costs tab under their real month, and the
   Budget Tracker / EOM / Costs figures reconcile (allowing for the valuation-date
   basis, which is by design).

The diagnostic endpoint pages/api/cost-reconcile.js can be deleted whenever you like.
