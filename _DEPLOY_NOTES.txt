WHAT'S NEW
----------
1) APPLICATION PDF (customer copy)
   One combined A4 PDF, Rock-branded, containing:
     • Header + application/valuation/payment/final dates
     • Summary (Measured Work / Variations / Materials On Site, Contract Sum /
       Application Total / Anticipated Final Account)
     • Payment Certificate (Gross / MCD / Sub-total / Retention / Total, across
       Current / Previously Cert / This Certificate)
     • Contract Works (measurable lines only; sub-headings/notes have no totals)
     • Variations (instructed only, with value-to-date)
     • Materials on Site — CUSTOMER COPY: shows the marked-up Total but NO mark-up
       % column (mark-up stays internal, exactly as agreed)
   Then the VARIATION and SUPPLIER-GROUP attachments are appended (PDFs copied in
   page-for-page; images placed on their own page).

2) DOWNLOAD
   "Download PDF" button in the application header (and inside the send popout).
   Opens /api/application-pdf?projectId=..&appId=..&download=1

3) SEND TO CUSTOMER
   "Send to customer" opens a popout (like the chase emails):
     • Tickable customer recipients (from the project's customer contacts / email),
       plus a free-text box for extra addresses
     • Editable subject + message (sensible default filled in)
     • Replies go to the project QS (reply-to) when set
     • Optional "Mark this application as sent" (freezes variations + locks it)
   Sends via Resend with the PDF attached, and records the send on the application.
   (Save any edits first — the Send button asks you to save if there are unsaved
   changes, so the PDF matches what's on screen.)
