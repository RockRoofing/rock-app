1) INVOICED / VAT / PAID — corrected definitions + why they didn't update
------------------------------------------------------------------------
Per your steer, the columns now mean:
  • Invoiced = TOTAL of all invoices on the system added together (inc VAT)
  • VAT      = Total VAT across all the project's invoices
  • Paid     = Total paid (Xero AmountPaid across all invoices)
(Previously "Invoiced" was showing ex-VAT — now it's the full total of all invoices.)

Why nothing updated after your sync: these figures are written by the wip-sync
Xero job, and are only captured once THIS code is deployed. Also the dashboard
caches results for 4 hours. So after deploying:
  → run the Xero WIP sync again (wip-sync). It rebuilds the invoice cache with
    the new totals AND clears the dashboard cache at the end, so the retention
    page will then show the figures.
If you sync and still see "—", the sync likely ran on the old code — re-run it
once this is live.

2) NEW "VAT TYPE" COLUMN (next to VAT)
--------------------------------------
Describes the VAT applied to the project's invoices, e.g. "20%", "5%",
"0% reverse charge", "0% zero-rated", "Exempt", or "Mixed" if invoices differ.
It's derived from Xero: reverse charge is detected from the invoice line TaxType;
otherwise the rate is inferred from VAT ÷ net (20/5/0%).

3) "RECEIVED" TICK NOT SAVING / NOT COLOURING — fixed
-----------------------------------------------------
Root cause: for a Xero-sourced row, its id was the Xero project id, so saving an
edit tried to UPDATE a manual record that didn't exist — the change (including the
Received tick) was silently dropped. Now, editing a Xero row saves a MANUAL
OVERRIDE keyed to that project (xeroId): the Received flags, dates and any edits
persist, the row shows your saved values, and the status/colour update correctly.
Editing the same row again updates that override rather than creating duplicates.
