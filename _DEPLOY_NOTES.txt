Retention register — all projects, Retention Owed + 612 columns, manual complete
================================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
(Builds on GroupC — deploy after/with it. retention.js here is the latest version.)

1) ALL IN-PROGRESS PROJECTS SHOWN
   The register no longer hides projects with nothing invoiced. Every project now
   appears; if nothing's invoiced, its retention values simply show £0. It's a full
   register.

2) NEW "RETENTION OWED" COLUMN
   Retention owed based on invoiced value = invoiced (Sales, account code 200) ×
   retention %. (Invoiced value already derives from the 200-Sales total, so this
   reconciles with the rest of the financials.)

3) NEW "612 ALLOCATED" COLUMN
   The retention actually deducted on invoices under account code 612, per project.
   • sync-invoices now captures 612 line amounts alongside 200.
   • dashboard surfaces it as retention612Allocated.
   ⚠ REQUIRES AN INVOICE RE-SYNC to populate (existing cached invoices don't have the
     612 figure yet). Until re-synced it shows £0.

4) MANUAL "MARK COMPLETE" — AUTO-COMPLETE REMOVED
   Each row has a "Mark complete" button (toggles to "✓ Complete"). Completion is now
   purely manual — the old auto-close (row auto-greened when Total Remaining hit £0)
   has been removed. The Outstanding / Complete / All filter keys off this manual
   flag, so Complete shows only projects you've marked complete.
