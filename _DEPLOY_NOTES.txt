RAMS — HOTFIX: "projectNo is not defined" when CM signs/approves
================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of fixes v4.

BUG
---
When the Contracts Manager tried to approve/sign a RAMS, it errored with
"project no. is not defined" (a JavaScript ReferenceError). Same would have hit
director-approve and set-site-manager.

CAUSE
-----
In the previous change that added the Site Manager "requires edits" (sm-reject)
action, the line that reads the project + file from the request —
  const { projectNo, fileId } = body
— was accidentally removed. The code just below it still used projectNo/fileId,
so those actions threw at runtime. (It compiled fine because the error only
occurs at execution, not at build.)

FIX
---
Restored the destructure before the cm-approve / director-approve /
set-site-manager actions. All three now have projectNo + fileId in scope again.
Verified with `node --check` and a clean build.

FILE CHANGED
------------
• pages/api/rams-approvals.js  — restored `const { projectNo, fileId } = body`

TEST
----
CM: open a project's RAMS → Approve as Contracts Manager → read to bottom →
sign + enter Site Manager → Approve & sign. Should now succeed (stage → Director).
