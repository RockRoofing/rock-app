=====================================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of batch 2d.

WHY PROJECTS WEREN'T PULLING OVER
---------------------------------
The Retention Tracker only listed projects that had already been INVOICED
(retentionOutstanding > 0 OR grossInvoiced > 0). Projects set up in your project
details (which feed the Budget Tracker / EOM report) but not yet invoiced were
filtered out — so they never appeared.

WHAT CHANGED
------------
• The tracker now shows every project that has a RETENTION % set in its project
  details, as well as any already invoiced / with retention outstanding. So the
  list mirrors the project-details data rather than waiting for invoicing.
• It now auto-fills from project details (via the same dashboard source the EOM
  report uses):
    – QS name (falls back to Estimator if QS not set)
    – Completion date (uses the project's completion date, else the PC date)
    – Retention comments (the box added in batch 1) flows into the row comments
  These layer on top of any manual override you've made, so they stay current.
• Dashboard API now exposes qsName, pcDate, defectsDate, completionDate and
  retentionComments from project settings.

IMPORTANT — clear the cache after deploy
----------------------------------------
The dashboard caches projects for 4 hours, so after deploying:
  → visit /api/clear-cache (admin) OR run the Xero WIP sync,
then reload the Retention Tracker. The extra projects + auto-filled fields appear.

If a project still doesn't show, check it has a Retention % set in Edit Project
Details → Retention. (A project with 0% retention has nothing to track, so it's
intentionally excluded.)
