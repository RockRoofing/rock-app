Weekly Report — editable recipients (portal users + manual) + editable day/time

Requires Vercel PRO (adds a 3rd, hourly cron).

WHAT'S NEW (all in the "Weekly Report" pop-out on Outstanding Invoices)
----------------------------------------------------------------------
• WHEN IT'S SENT: pick the day of week + hour (e.g. Thursday at 15:00). UK time,
  sends on the hour. Saved as you change it.
• PORTAL USERS: a checklist of all active portal users — tick anyone who should
  receive the report.
• OTHER EMAIL ADDRESSES: add/remove any manual email (e.g. someone outside the
  business), as before.
• Send now: unchanged.

RECIPIENT MODEL CHANGE (please note)
------------------------------------
The report now goes to EXACTLY the people in this pop-out (ticked portal users +
manual emails). Previously it auto-sent to everyone with post-contract/mgmt/admin
access. Safety net: if the list is ever empty, it falls back to the old auto
post-contract/mgmt/admin recipients so it never sends to nobody. => After
deploying, open the pop-out once and tick who should receive it.

HOW THE TIMING WORKS (honest detail)
------------------------------------
A once-a-day cron can't fire at an arbitrary hour, so this adds a dedicated cron
that runs HOURLY (/api/cron/weekly-report) and sends only when the configured
day + hour match the current UK time, guarded so it never sends twice in a day.
The old Thursday piggyback inside hs-expiry-email has been REMOVED (no double
sends). vercel.json now has 3 crons (Pro allows this):
   deliveries-sync  0 6 * * *
   hs-expiry-email  0 8 * * *
   weekly-report    0 * * * *   (NEW, hourly)
Test immediately with:  /api/cron/weekly-report?force=1
