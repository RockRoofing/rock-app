Project Financials — always use LIVE WIP override/adjustments/dates
===================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: Compiled successfully
Single file: pages/api/dashboard.js
Deploy AFTER the PF_mirror_wip_override and WIP_shared_calc_sync bundles.

WHY PF STILL DIDN'T MATCH
-------------------------
PF uses the SAME shared WIP calculation as the WIP page, with the SAME cost/invoice
lines. The mismatch was the INPUTS coming from the 4-hour dashboard cache: the WIP
margin override, the manual WIP adjustments, and the per-month valuation-date
overrides could be stale in the cache even though the WIP page reads them live. So the
formula was identical but PF was feeding it old override/adjustment/date values.

THE FIX
-------
When PF serves from the dashboard cache, it now OVERLAYS these three WIP-relevant
fields from live storage for every project:
  • wipMarginOverride   (from live project settings)
  • dateOverrides       (from live project settings — drives the valuation date)
  • wipAdjustments      (from live wip:adjustments:{id})
Everything else still comes from the warm cache (fast), but the WIP inputs are always
current — so PF now mirrors the WIP page immediately, without waiting for a full cache
rebuild.
