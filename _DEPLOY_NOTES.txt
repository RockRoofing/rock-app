Commercial Scorecard — Payless Notices = credit notes (adjustable + drill)
==========================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: Compiled successfully
Deploy AFTER the credit-notes sync fix (so credit notes exist in invoiced:lines).

WHAT CHANGED
------------
Payless Notices now = the number of CREDIT NOTES applied, counted by month. Every
credit note is one payless notice (previously it was underpayments / a flag).

• The card shows the ADJUSTED total across months (with "N raw" underneath when any
  month has been manually adjusted).
• Click the card to open a month-by-month view. For each month it shows the raw credit
  note count and an editable "counts as" number so you can adjust it (e.g. 7 raw ->
  4 because 3 were minor). Save persists it; Reset clears back to the raw count.
• Under each month, the credit notes that make up the number are listed: project
  (job no + name), credit note number, the invoice it was applied to, date and amount.
• The trend sparkline uses the adjusted monthly count.

DATA
----
• Reads credit notes from invoiced:lines:{id} (creditNote flag) per live project.
• Manual monthly adjustments saved to config:payless-adjustments  { 'YYYY-MM': count }.
• POST /api/commercial-metrics { month, adjusted } saves an adjustment (adjusted=null
  clears it).

FILES
-----
• pages/api/commercial-metrics.js   (credit-note payless + per-month counts + POST)
• pages/commercial-scorecard.js     (adjusted card value, PaylessModal drill + editor)

NOTE: "Applied to invoice" shows if that link is stored on the credit note; some Xero
credit notes don't carry an allocation reference, in which case it shows the CN number
/ reference only.
