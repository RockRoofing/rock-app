WHY STILL 401
-------------
The sync refreshes the Xero token and uses the SAME token for the P&L (works) and the
Journals endpoint (401). So the token itself lacks the journals permission. Two causes:
  (A) The reconnect didn't actually grant it (old refresh token still in use / consent
      not completed with the new scope), OR
  (B) The scope isn't enabled on the Xero Developer APP, so Xero won't grant it even
      when asked.

This build shows EXACTLY which, by printing the current token's granted scope.
