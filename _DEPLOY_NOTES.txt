Applications: orange variations + PO mark-up  ·  Delivery auto-assign fix
========================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully  ·  Applications tab shows "· v5"

1) NOT-INSTRUCTED VARIATIONS = ORANGE
   The Not-instructed status pill is now orange (same shape/weight as the green
   Instructed pill). Still greyed line text + N/A value-to-date + excluded from
   totals; the pill itself is orange for visibility. Toggling still writes to the
   tracker/budgets.

2) MATERIALS ON SITE — ALL POs + PER-PO MARK-UP
   • "+ Add from POs" now lists ALL purchase orders for the project (delivered or
     not; each PO is tagged Delivered / Not delivered), grouped by PO, showing
     each PO's line items.
   • Each PO has a Mark-up % — applied to every line added from that PO. You can
     "Add all lines" for a PO or add individual lines. A manual line can also take
     a mark-up.
   • On the application, Materials on Site now shows Net, a Mark-up % column
     (shaded, editable), and Total = net x (1 + mark-up). The value used
     everywhere (Summary/Certificate) is the MARKED-UP total.
   • CUSTOMER COPY: the mark-up % column is internal only and will be HIDDEN on the
     customer PDF (Phase 4); the marked-up Total is what the customer sees. (Per
     your choice: both show the marked-up value; the PDF just hides the % column.)

3) DELIVERY SCHEDULE — AUTO-ASSIGN PROJECT ON RE-SYNC  (bug fix)
   Previously, re-syncing skipped POs already known, so if you added the Xero
   project (tracking) category to a PO AFTER it was first synced, the delivery
   never picked it up. Now a re-sync UPDATES an existing delivery's project from
   the PO's current Xero tracking category when it's been added or changed. New
   POs still auto-assign from the category as before. (sync result now also
   reports "updated".)
