Chase Emails — PHASE 2: timeline + compose/send popup
=====================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Deploy Phase 1 first (needs lib/chaseEmailTemplates.js + the templates API).

WHAT'S NEW
----------
• Auto Send column now shows a TIMELINE of 5 dots per invoice:
  U · 1 · 2 · 3 · W  (Upcoming, Overdue 1/2/3, Withdrawal).
  Grey = not sent, green/filled = sent. Click ANY dot in any order to compose
  that email. Sending a later one does NOT change earlier dots (no dropping
  back); a sent dot can be clicked again to resend.
• Clicking a dot opens a COMPOSE POPUP: subject + body pre-filled from that
  stage's template with [merge fields] resolved for the invoice, fully editable
  before sending. Includes a discreet "Amend template ↗" link (to the Commercial
  editor) and a Cancel/Send.
• Sends via Resend through NEW /api/chase-email-send:
    FROM  = ACCOUNTS_FROM_EMAIL  (falls back to FORMS_FROM_EMAIL, then the Resend
            test address)
    REPLY-TO = the project QS email (row.qsEmail), so replies reach the QS.
• On send, the stage is recorded on the invoice (new 'record-chase' action in
  /api/outstanding-invoices, stored in the invoice meta as chases{stageKey:{at,
  to,subject}}). The timeline reflects it immediately and after reload.

MERGE FIELDS resolved now: First Name (from customer name), Company Name,
Invoice Number/Reference, Project Name, Due Date, Today's Date, Invoice Value,
Rock Roofing QS Name (project QS, or the logged-in user). Fields we don't yet
hold on the invoice row (Customer Address, Customer QS Name/Email, Project
Address, Sub-Contract Ref) resolve blank — the QS can type them in the editable
popup. We can wire these to real data in a later pass if you store them.

*** ENV / SETUP (required for real sending) ***
1. Set ACCOUNTS_FROM_EMAIL in Vercel, e.g.:
     Rock Roofing Accounts <accountsreceivable@rockroofing.co.uk>
   (Left unset, it falls back to FORMS_FROM_EMAIL / the Resend test address.)
2. Verify the ROOT domain rockroofing.co.uk in Resend (add its SPF/DKIM/return-
   path DNS records). Your existing notifications use the verified SUBdomain
   send.rockroofing.co.uk; sending from accountsreceivable@rockroofing.co.uk
   (root) needs the root verified too. Until then, use an @send.rockroofing.co.uk
   address in ACCOUNTS_FROM_EMAIL, or the Resend test address.
