WHAT THIS DOES
--------------
Reintroduces the nightly incremental sync, now capturing ALL THREE sources:
  • Bills (ACCPAY)        -> materials + subcontractor labour (excl. 320)
  • Direct Wages (320)    -> from Manual Journals, project-tagged lines only
  • Sales Invoices (ACCREC) -> invoiced / paid / due
It rotates ONE project per night (deep-sync:pointer) and, for that project, tops
up the last 120 days from Xero. So the full cycle is one project/night — over a
few weeks every project is refreshed, then it loops.

MODEL (important)
-----------------
Upload first, then let the cron top up:
  1. Upload Bills + Direct Wages + Sales Invoices (Admin -> Xero Upload) to seed
     the full history now.
  2. The nightly cron then only ADDS items that appeared in Xero since, within a
     rolling 120-day window.

DE-DUPLICATION (your question)
------------------------------
Dedupe = stopping the same transaction being counted twice. Each line gets a
fingerprint (date + amount + account + description; invoices by invoice number).
Before adding a line from Xero, the sync checks whether that fingerprint is
already stored for the project — if so it's skipped. This is what makes the
120-day overlap safe: recent transactions are re-scanned every night to catch
late-posted ones, and dedupe ensures they only ever count once. Costs are written
to the per-source keys (costs:bills / costs:wages) and summed via mergeCosts, so
the sync never clobbers your uploaded figures — it adds to them.
