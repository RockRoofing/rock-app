Financial App — batch 3b (update): fix input focus bug + Project Name autocomplete
==================================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Replaces the batch 3b pages/retention.js (deploy over the top).

1) FIXED: form boxes losing focus after each character
------------------------------------------------------
The Add/Edit form was defined *inside* the page component, so every keystroke
recreated it and React threw away the input (kicking you out after one letter).
The form is now a top-level component, so inputs keep focus — you can type
normally.

2) "Add existing project" is now an autocomplete on Project Name
---------------------------------------------------------------
The separate project picker dropdown is gone. Instead, when adding a NEW entry,
start typing in the Project Name box (e.g. "Far") and matching existing projects
appear beneath it (matches on project name, ref or customer). Pick one to
auto-fill Our Ref, Customer, Project Name, Project Value, Final Account,
Retention %, Completion Date and QS — and link it so comments sync back.
Typing a fresh name that you don't pick from the list = a normal manual entry.
The suggestions close when you click away.
