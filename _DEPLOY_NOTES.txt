WHY
---
A full live Xero sync of every invoice hits Xero's rate limit and Vercel's 10s
function timeout. Instead, you export "Sales Invoices" from Xero to CSV and upload
it here — the app parses it and fills Invoiced / VAT / VAT Type / Paid / Due for
ALL projects in one go, with essentially no Xero API calls.

HOW TO USE
----------
1) In Xero: Business → Invoices → export the Sales Invoices list to CSV (the file
   with columns InvoiceNumber, Total, TaxTotal, InvoiceAmountPaid,
   InvoiceAmountDue, TrackingOption1, TaxType, Status, etc. — i.e. exactly the
   export you sent me).
2) In the app: Retention Tracker → "⬆ Import invoices from Xero" → choose the CSV.
3) It reports how many invoices/projects it imported (and lists any project names
   in the file that didn't match an app project). The tracker refreshes with the
   figures.

WHAT IT DOES (verified on your sample)
--------------------------------------
• Reads the line-level export, de-duplicates invoices by InvoiceNumber (Total/
  Paid/Due are invoice-level and repeat on each line — they are NOT summed per
  line), and aggregates per project by TrackingOption1 (e.g. "J228-Farmstead
  Drive").
• Skips overpayments/prepayments; counts sales invoices, subtracts credit notes.
• Captures VAT (TaxTotal) and a VAT-type label per project: "20%", "5%",
  "0% reverse charge", "0% zero-rated", "No VAT", "Exempt", or "Mixed" when a
  project's invoices differ (your J228 sample = Mixed: reverse-charge + zero-rated).
• Matches each tracking name to the app's project via one lightweight Xero call
  (tracking categories only — not the rate-limited per-invoice calls), then writes
  the same cache the Retention Tracker & dashboard read, and clears the dashboard
  cache so figures show immediately.
  (Sample result: J228-Farmstead Drive — 8 invoices, £92,389.85 invoiced,
   £75,226.55 paid, £17,163.30 due, £0 VAT.)

NOTES
-----
• If a project name in the CSV doesn't match an app project, it's listed as
  skipped (nothing is silently lost) — usually a naming mismatch we can align.
• Re-importing simply overwrites with the latest snapshot (safe to run repeatedly).
• I also made the live Xero retry logic honour Xero's Retry-After header (helps
  the daily sync recover from throttling), but the CSV import is the reliable
  route for bulk data.
