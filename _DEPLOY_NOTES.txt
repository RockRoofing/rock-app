RAMS SIGNING — PHASE 2: Operative signing
=========================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully (Xero warnings unrelated)

WHAT THIS DOES
--------------
Operatives can now SIGN a RAMS in the Site App (siteapp.rockroofing.co.uk).

In the RAMS folder, each unsigned RAMS shows a red "● Not signed" status and a
red "✍ Sign RAMS now" button. Tapping it opens a full-screen flow:

  1. READ — the RAMS PDF renders on screen (pdf.js canvas, mobile-safe). The
     "continue" button stays disabled until the operative scrolls to the BOTTOM
     of the document ("↓ Keep scrolling" turns into "✓ You've reached the end").
     Short docs that don't scroll are auto-counted as read.
  2. CONFIRM & SIGN — the exact RAMS statement is shown; Name (logged-in
     operative) and Date are auto-filled; a finger/mouse SIGNATURE PAD lets them
     draw their signature (Clear to redo).
  3. "Sign & confirm" saves the signature. The row flips to green "✓ Signed"
     immediately, and the home RAMS badge decrements when they return home.

The statement signed (stored with each signature):
  "I confirm I have read, fully understood and will work to this and any other
   documents relating to this method statement. If at any point I feel it is
   unsafe to continue I will stop works and contact my supervisor. Any
   amendments to this method statement must be made by the person who originally
   completed it. It must then be communicated to the relevant persons."

VERSION RESET (built-in, no extra logic)
----------------------------------------
Signatures are keyed by the RAMS file id. Uploading a new RAMS to the project
mints a new file id (via /api/project-files), so the new version shows as
"Not signed" for everyone and the badge/notification reappears automatically.
No separate version-detection needed.

DATA
----
Each signature stored at ops:rams-signatures:<projectNo>[fileId][opId] =
  { name, date, signedAt, statement, signatureImg }
signatureImg is a PNG data-URL of the drawn signature — Phase 5 uses this to
build the portal signature page appended to each RAMS.
