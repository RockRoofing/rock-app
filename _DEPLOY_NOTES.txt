WIP page fixes
==============
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Deploy AFTER the initial WIP_page bundle.

FIXES
-----
1. PROJECT NAMES showed as raw IDs (abca7170-…). Cause: names live on the Xero
   tracking categories, not the settings object. The WIP API now resolves each
   project's name + job number from the dashboard cache (keyed by xeroId). Projects
   that can't resolve to a real name are skipped (no more raw GUIDs).
   → Make sure the dashboard has been loaded/synced at least once so the cache exists.

2. COSTS SORTED BY DATE by default, and every column header in BOTH tables (costs and
   credit notes) is now clickable to sort (asc/desc, with an arrow indicator).

3. MARGIN is now editable per project — a % box + Save in each project header. Saving
   stores it as the project's wipMarginOverride and recomputes that project's WIP.

4. TOTAL WIP at the top does NOT deduct credit notes. It is costs + margin + manual
   adjustments only (WIP = (post-val costs + manual adjustments) ÷ (1 − margin)).
   Credit notes are shown on the right for information and never reduce the total.

ALSO
----
• pages/api/project/[id]/settings.js now MERGES the posted fields with existing
  settings (instead of overwriting), so saving just the margin from the WIP page
  can't wipe other project settings. Full-object callers are unaffected.
