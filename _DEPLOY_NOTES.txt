Bookkeeping — DIAGNOSTIC build for empty Direct Wages tab
=========================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy over the top.

WHY
---
The Direct Wages tab shows nothing (categorised or uncategorised). Code trace
shows the data path is correct (importer -> costs:wages:<id> -> mergeCosts ->
costs:lines:<id>; API reads costs:lines:<id> filtered to 320 + costs:untagged:wages).
So the likely cause is empty stores or an empty project list — not a display bug.

WHAT THIS ADDS
--------------
The /api/bookkeeping response now includes a "_diag" object:
  • projectsRead        — how many projects the API read (0 = project list empty)
  • projectListEmpty    — true if it couldn't resolve any projects
  • untaggedWagesRaw    — rows in costs:untagged:wages
  • untaggedBillsRaw    — rows in costs:untagged:bills
  • categorisedWageLines— 320-coded lines found across projects
  • totalCostLines      — all cost lines read across projects
  • wagesTabCount       — final count on the Direct Wages tab

HOW TO READ IT
--------------
Open (while logged in as admin) in the browser:
    https://app.rockroofing.co.uk/api/bookkeeping
and look at "_diag". Tell me the numbers. Likely outcomes:
  • projectListEmpty=true  -> project list couldn't load (dashboard cache empty +
    Xero token issue). Fix: view the Commercial/dashboard once, or we cache the list.
  • projectsRead>0 but categorisedWageLines=0 and untaggedWagesRaw=0 -> no wages in
    the stores; the last wages upload didn't land / needs re-upload.
  • untaggedWagesRaw>0 but wagesTabCount=0 -> a display bug (unlikely) — tell me.

No behaviour change other than the extra diagnostic field.
