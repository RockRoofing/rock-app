THE "NO WIP TO SHOW" BUG
------------------------
The WIP page skipped EVERY project when it couldn't resolve a project name, and name
resolution depended entirely on a fresh dashboard cache. If the cache was stale/empty
(which the recent cache-guard change can cause on first load), every project was
dropped -> "No WIP to show for <any month>".

FIX: the WIP API now resolves project names/job numbers straight from the Xero
tracking categories (the authoritative source, same as the dashboard), falling back
to the dashboard cache. It no longer depends on the cache being fresh, so projects
show again. (Was also using the wrong field name — trackingOptionId, not xeroId.)

PER-LINE MARGINS (your point 3) — ALREADY THE BEHAVIOUR
------------------------------------------------------
Each manual WIP adjustment is grossed at ITS OWN margin; post-valuation COSTS are
grossed at the overall project (margin-adjusted) margin. So:
  • Soprema invoice entered at 0%  -> adds exactly £11,000.
  • A cost of £3,171 at the project's 14.4% -> £3,704.
No change needed here; it works as requested. (The earlier £932 came from the
cancelling adjustment being at −14.4% instead of +14.4%.)
