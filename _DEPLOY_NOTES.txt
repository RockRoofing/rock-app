ISSUE
-----
Home Deliveries badge showed 3 but only 1 was overdue when you opened the
project. Cause: the badge counted ALL not-yet-delivered deliveries, while the
project view filters to a 2-week look-ahead — different rules.

FIX (per your steer)
--------------------
Both the home Deliveries badge and the per-project badges in the Deliveries
page now count ONLY deliveries whose scheduled/required delivery date is
TODAY or earlier and not yet delivered (i.e. overdue + due today).
  • Home badge  = total such deliveries across the operative's projects.
  • Per-project = that project's count.

The in-project list still shows the useful 2-week look-ahead so operatives can
see what's coming, BUT overdue / due-today rows are now clearly tagged in red
("OVERDUE" / "DUE TODAY") so the list visibly reconciles with the badge count.
