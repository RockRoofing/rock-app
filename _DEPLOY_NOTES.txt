DE-DUPLICATION NOW USES UNIQUE IDENTIFIERS (your suggestion)
-----------------------------------------------------------
Instead of a fuzzy fingerprint, dedupe now keys on the most reliable identifier
available:

• Sales invoices → INVOICE NUMBER (unique in Xero). The sync also stores the
  Xero InvoiceID. Matches uploaded invoices (which carry the invoice number).
• Bills → the Xero LineItemID (true GUID) when present, else invoice number +
  account code + amount. The sync stores InvoiceID + LineItemID on each line.
• Direct Wages → the Xero JournalLineID (true GUID) when present, else
  date + amount + account 320. The sync stores ManualJournalID + JournalLineID.

Why the fallbacks: the Xero CSV/Excel EXPORTS you upload do NOT contain the GUIDs
(only invoice/journal numbers), so a pure-GUID key couldn't match uploaded rows.
The sync stores the real GUIDs it gets from the API (so sync-to-sync dedupe is
exact), and falls back to the business key to line up with uploaded rows. Net
effect: no transaction is counted twice, whether it came from an upload or the API.
