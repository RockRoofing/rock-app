RAMS — Customer gets a copy of the RAMS when they approve
=========================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Requires Phase 5 (lib/signedRamsPdf.js) to be deployed. Deploy on top of it.

WHAT CHANGED
------------
When the customer's Site Manager approves the RAMS (via the no-login link), they
now receive a confirmation email — "Your approved RAMS — <project>" — with a COPY
of the RAMS attached for their records.

The attached copy is the SIGNED version: the original RAMS document followed by
the approval record (Contracts Manager, Director and their own approval, with
names, dates and times). If the signed build can't be produced for any reason, it
falls back to attaching the plain original RAMS so the customer always gets a copy.

The email is sent AFTER their approval is recorded, so the approval record in the
attachment includes their sign-off. Operatives are still notified to sign as
before; this is an additional email to the Site Manager only.

FILE CHANGED
------------
• pages/api/rams-approvals.js — sm-approve now calls sendSiteManagerApprovedCopy()
  (new helper) which builds the signed PDF and emails it to the Site Manager.

NOTE
----
Needs RESEND_API_KEY + FORMS_FROM_EMAIL set in Vercel (as with all RAMS emails).
Best-effort: approval still succeeds even if the copy email can't send.
