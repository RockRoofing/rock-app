Deploy over the top.

1) TAB RENAMED
   "Ignored" tab is now labelled "Overheads". (Data/plumbing unchanged — it still
   shows Ignore-category costs + untagged overheads.)

2) NIGHTLY SYNC NOW FEEDS OVERHEADS / UNTAGGED INTO BOOKKEEPING
   Previously the nightly deep-sync only captured project-tagged cost-of-sale
   lines, so the Overheads/untagged view only stayed current via manual uploads.
   Now the sync also runs a once-per-day "untagged bills sweep":
     • Pulls ALL ACCPAY bills in the 120-day window
     • Keeps lines with NO project tracking tag (overheads + untagged cost-of-sale)
     • Merges them into costs:untagged:bills (deduped by Xero LineItemID / business key)
   So the Overheads tab now stays current automatically.

   API COST: this is heavier than the per-project fetch (it reads all bills, not
   just one project's), which is why it runs ONCE PER DAY total (guarded by
   'untagged-sweep:at'), not on every project rotation. It reuses the same 120-day
   window + dedupe, so it won't double-count. Still well within Xero's daily limit,
   but it's the most API-intensive part of the sync — worth watching the first few
   runs (the response now reports 'untaggedAdded').

   NOTE: untagged WAGES are not swept (wage journals are project-tagged transfers);
   overhead capture is for bills. Manual wage uploads still feed the wages side.
