ADMIN DOCUMENTS — fix upload error + single-file UX + in-app viewing

THE ERROR ("Unexpected token 'R', 'Request En'... is not valid JSON")
  Cause: the photo was bigger than Vercel's ~4.5MB serverless request-body limit, so /api/upload-file got
  a plain-text "Request Entity Too Large" response (not JSON) and the client's r.json() choked.
  Fix: uploads now go DIRECT from the browser to Vercel Blob (client upload), which bypasses that limit.

  pages/api/blob-upload.js  (NEW) — token endpoint for direct browser->Blob uploads (handleUpload).
    Allows images / PDF / Word / Excel / text up to 50MB.

  pages/admin/documents.js
   - Uses the client upload() flow (no size error on big photos/docs).
   - ONE file at a time: "Choose file" -> shows the filename with an × to CLEAR it -> "Upload".
   - Upload is disabled until a file is chosen. After a successful upload the file + name clear.
   - Each uploaded card still has Delete (with confirm) and inline rename, as before.

SITE APP — documents now viewable IN-APP
  pages/forms/browse.js
   - Company Information / Operative Guidance cards now open in the in-app viewer (pdf.js canvas for PDFs,
     image viewer for images) with prev/next, instead of opening the raw file URL in a new tab (which
     mobile browsers refuse to render inline).

Deploy all files. No cron/env change (uses the existing BLOB_READ_WRITE_TOKEN).
