PLANNING — fix: can't edit existing/historic allocations + marker tweaks

  pages/operations/project-management/planning.js

  BUG FIX (the real cause of "no way to edit historic/existing allocations"):
    The "Edit labour allocation" button is shown when the selected cells already have labour. That check
    used (cell || []).length — but cells are now stored as objects { status, unnamed, entries }, so
    .length was undefined and the button NEVER appeared for any allocation saved in the new format.
    Fixed to use cellData(cell).count. Now selecting any cell(s) with labour (past OR future) shows
    "Edit labour allocation", which opens the pop-up pre-loaded with the current people/headcount/status.
    (Historic editing works once "Historic" is toggled on so past days are visible.)

  MARKERS (per request):
    - Removed the blue current-week (W/C) line.
    - Today marker is now a single dark-green line on the RIGHT of today's cell only.
    - Legend updated to match.

Deploy over previous planning.js. No API change.

TO EDIT PAST LABOUR: toggle "Historic" (shows last 2 weeks) -> select the past cell(s) -> click
"Edit labour allocation" -> change people/headcount, and set "Actual" if the dates have passed -> Save.
