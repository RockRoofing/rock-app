REPORT A PROBLEM — portal (every page) + Site App, with an admin viewer

NEW API — pages/api/report-problem.js
  POST  { userName, platform, page, description } -> stores under ops:problem-reports + emails the office
        (to ALERT_EMAIL or FORMS_REPLY_TO if set; email is best-effort and never blocks saving).
  GET   -> { reports } newest first.
  PATCH { id, status } -> open/resolved toggle.

PORTAL — every page (components/ReportProblemButton.js + pages/_app.js)
  - A small "⚠ Report a problem" button is fixed top-right on every PORTAL page (auto-hidden on the Site
    App). Opens a popup that AUTO-fills the user's name (from portal-auth) and platform (Portal), plus the
    current page path (editable) and a description box.

SITE APP — bottom of the operative menu (pages/forms/index.js)
  - A "⚠ Report a problem with the app" link at the bottom of the Site App home. Popup auto-fills the
    logged-in operative's name and platform (Site App), plus a page box and description.

ADMIN VIEWER — pages/admin/problem-reports.js + components/AdminShell.js
  - New "Problem Reports" tab in the Admin area: lists reports (Open / Resolved / All), shows who/where/
    page/description, with a "Mark resolved / Reopen" toggle.

OPTIONAL ENV: set ALERT_EMAIL to control where the notification email goes (falls back to FORMS_REPLY_TO,
then notifications@rockroofing.co.uk). Not required — reports are always stored and viewable in Admin.

Deploy all files. No cron change.
