WHAT IT DOES
------------
Read-only endpoint to explain WHY the Budget Tracker, EOM Report and Project page
can show different labour/materials/total for the same project.

USE IT
------
Deploy, then open in your browser while logged in (find Russell Hill's Xero id — it's
in the project page URL /project/<id>):

  https://app.rockroofing.co.uk/api/cost-reconcile?id=<xeroId>&date=2027-06-30

IT RETURNS
----------
• cached_costs_latest        — what the BUDGET TRACKER shows (all-time, import-time
                               labour/materials split)
• recomputed_all_time_320_321 — same lines, all-time, labour = codes 320/321 only
• recomputed_to_date          — same lines up to the ?date you pass (what the PROJECT
                               page / EOM compute)
• classification_check        — labour by the import 'type' field vs labour by code
                               320/321. If 'difference' is NON-ZERO, some spend is
                               called Labour by the categorisation config but isn't
                               coded 320/321 — that's a real classification mismatch,
                               separate from the date cutoff.
• account_code_breakdown      — every account code present with its total, so you can
                               see exactly which codes make up labour vs materials.
