THE BUG
-------
The rebuilt cards referenced `onPickPL` (the future P&L drill-down handler) in the
JSX, but it wasn't declared as a ReconPanel parameter. Referencing an undeclared
VARIABLE throws a ReferenceError (unlike an undefined object property), which
crashed the whole Bookkeeping page: "onPickPL is not defined".

THE FIX
-------
Added onPickPL to the ReconPanel parameter list. It's still not passed by the
caller (drill-down is Phase 2), so it's simply undefined — the `onPickPL && ...`
guards now evaluate safely and the grey "P&L (Xero): £X" line shows WITHOUT the
"investigate" link, exactly as intended for this phase.
