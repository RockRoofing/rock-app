THE BETTER APPROACH (your steer)
--------------------------------
Instead of reconstructing "net incl retention" as (ex-VAT subtotal + retention),
we now sum the invoice lines coded 200 (Sales) directly. On your invoices the 200
line carries the FULL work value net of VAT WITH retention still in it; retention
is posted to a separate account (612), so the 200 total already includes it. No
adding retention back on — more accurate, and it moves correctly when you edit
invoices in Xero.

CHANGES
-------
• pages/api/sync-invoices.js — while reading each invoice's line items (already
  fetched for tracking), also sum lines where AccountCode==200 -> per-invoice
  `sales200`, aggregated to `invoicedSales200` per project and stored.
• pages/api/dashboard.js — uses invoicedSales200 as the invoiced figure:
    - grossInvoiced   = invoicedSales200            (net of VAT, incl retention)
    - remainingToClaim = afa − grossInvoiced
    - retention amounts derived for display (totalRetention = sales200 × pct)
    - "fully invoiced/closed" check compares grossInvoiced vs afa
  Falls back to the previous ex-VAT+retention method if 200 data isn't present
  yet (i.e. before the first re-sync).
• pages/retention.js — "Invoiced Net" column RENAMED to "Invoiced"; its value is
  now the 200-based figure (grossInvoiced). Header tooltip explains: sum of
  account-code-200 Sales lines, net of VAT, including retention.
• pages/commercial.js — Budget Tracker "Gross Invoiced" column RENAMED to
  "Invoiced" with a hover tooltip describing the same (net of VAT, incl retention,
  from code 200). Column-customiser label updated too.

WHY WINNERSH WASN'T UPDATING (your report)
------------------------------------------
The retention MERGE logic was correct (it layers live figures over saved entries),
but the figure it layered was the old ex-VAT reconstruction. With the 200-based
value, editing Winnersh invoices in Xero and running Sync Invoices will now flow
through to both pages. If a figure still looks old after syncing: check the sync
WINDOW covers the invoice dates (bump to 12/24 mo if the invoices are older), and
hard-refresh so the 4h dashboard cache (which the sync clears) is re-read.
