NEW ADMIN AREA: Account Categorisation  (/admin/account-categorisation)
----------------------------------------------------------------------
New tab in the Admin sub-nav (after RAMS Director). Admin-only.

• Lists all cost-of-sale account codes + names.
• For each account, choose: LABOUR / MATERIALS / IGNORE (ignore = excluded from
  project costs). This controls the Labour vs Materials split across Project
  Financials — it's now configurable instead of hardcoded.
• The standard cost accounts (310, 311, 320 Direct Wages, 321 CIS Labour, 322,
  325, 328–336, etc.) show immediately with sensible defaults (320/321 = Labour,
  the rest = Materials). Any OTHER account that appears in your Xero cost export
  is auto-added to this list after the next upload, so nothing is hidden.
• Save, then re-upload a project's Cost Transactions to recalc its split.

IMPORTER CHANGES (pages/api/upload-transactions.js)
---------------------------------------------------
• Now recognises ANY account header in the Account Transactions export — not just
  a hardcoded list — so new/unmapped accounts are captured (previously silently
  skipped).
• Records every account it sees (code -> name) so the admin page can list them.
• Uses the admin categorisation config for the Labour/Materials/Ignore decision,
  falling back to the old default (320/321 = labour, else materials) when an
  account hasn't been categorised yet.
