FIX: chase emails not sending / not logging to comments  +  To-box UI change
============================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Deploy on top of the existing chase-email + ProjectPeople work.

ROOT CAUSE OF THE BUG
---------------------
Three endpoints guarded access with roles ['post-contract','management','admin'].
The role model now also has an 'accounts' role — and the person chasing invoices
is on that role. So:
  • /api/chase-email-send      → silent 403 → the email never sent.
  • /api/outstanding-invoices  → silent 403 → record-chase (the comment) AND
    even add-comment were blocked → nothing saved to the invoice comments.
  • /api/chase-email-templates → 403 → couldn't load/save templates.

FIX
---
• Added 'accounts' to the allowed roles on all three endpoints.
• The compose box now SHOWS a failure if logging the sent email to comments
  fails (previously swallowed): "Email sent, but logging it failed: …".

If sending still fails after this, the message shown in the compose box will say
why — most likely a Resend/env issue (RESEND_API_KEY or ACCOUNTS_FROM_EMAIL).

UI CHANGE (as requested)
------------------------
• The "To" box and the "Hi [name]" greeting now start BLANK when the compose
  popup opens.
• "To" is a DROPDOWN OF ALL CUSTOMER CONTACTS for that project (from the IHM),
  single recipient — plus a free-type box to enter any email.
• Selecting a To contact AUTO-FILLS "Hi <first name>," in the body.
• If you free-type an unknown email, the greeting is left BLANK for you to fill.
• CC remains multiple (unchanged).
