ATTACHMENTS — View button fix (render viewer via portal)

  components/RowAttachments.js
   - Root cause: the in-app viewer overlay was rendered from inside a table cell. A position:fixed element
     inside a table/scroll container whose ancestor has a transform (or similar) gets clipped/hidden, so
     "View" appeared to do nothing — while "Download" (a pure JS action) worked fine.
   - Fix: the viewer overlay is now rendered through a React portal to document.body, so it always covers
     the full viewport regardless of the table's layout/stacking context. Raised its z-index to 3000.
   - "View" no longer closes the attachments list first; the viewer simply layers on top.
