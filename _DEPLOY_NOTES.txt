Valuation date dropdown — exclude FUTURE override dates (was defaulting to 2027)
===============================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Single file: pages/project/[id].js

THE MISMATCH YOU SAW
--------------------
Russell Hill's project page defaulted to valuation date 30 JUNE 2027 (a future date),
while the EOM report was on June 2026 — a full year apart — so every figure differed
(more costs/invoices fall within the 2027 cutoff).

ROOT CAUSE
----------
getPastValuationDates() added manual per-month override dates WITHOUT a future check
(the fixed-day branch had one, the override branch didn't). Russell Hill has a
30 Jun 2027 override row, which sorted to the top of the "most recent first" list and
became the default selection.

THE FIX
-------
Future override dates are now excluded from the valuation-date dropdown, so it only
offers dates up to today and DEFAULTS to the most recent PAST valuation date. Russell
Hill will now default to its latest past date (e.g. 30 Jun 2026), which lines up with
the EOM report's month — and the figures reconcile.

(Pairs with the earlier lib/db.js fix, which made the server-side effective valuation
date also use the most recent past override.)

WORTH CHECKING IN THE DATA
--------------------------
Russell Hill has a valuation-date override entered for June 2027. If that's a typo
(should be 2026), correct it in Edit Project Details > monthly date overrides. The
app now handles it correctly either way, but a stray 2027 date won't be used until
that month actually arrives.
