FIX: Weekly report (with comments) PDF download failed on emoji
===============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully

THE ERROR
---------
Download returned: {"error":"WinAnsi cannot encode \"\" (0x1f4e7)"}
0x1f4e7 is the 📧 emoji we put at the start of logged chase-email comments.
The PDF uses standard fonts (Helvetica / WinAnsi encoding), which can't encode
emoji or various unicode symbols — so the whole PDF build threw and the download
failed.

THE FIX
-------
Added a text sanitiser applied to every string drawn into the PDF:
  • 📧  → "[email]"
  • smart quotes “ ” ‘ ’ → " '
  • en/em dash – —      → -
  • ellipsis …          → ...
  • bullet •            → *
  • anything else outside WinAnsi (incl. other emoji like 🙏) is dropped.
Verified: after sanitising, zero characters remain that WinAnsi can't encode, so
the report (including the full email bodies now stored in comments) generates
cleanly. Content still reads fine; only unsupported glyphs are swapped/removed.
