Upcoming Applications — app + valuation dates, overdue drafts
=============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully  ·  Applications tab shows "· v18"

1) COLUMNS: APPLICATION DUE + VALUATION DATE (replaces Month)
   The table now shows "Application due" (colour-coded: grey not-yet-due / orange
   within 3 days / green due today / red overdue, with in-Xd / Xd-overdue) and a
   "Valuation date" column. The Month column is removed.

2) OVERDUE DRAFTS ALWAYS SHOW
   A draft application that hasn't been marked as sent and whose application date
   has already passed now always appears in the table (previously it dropped off
   because it was outside the 31-day forward window). Overdue drafts use their own
   saved application/valuation dates, so the date shown is correct and flagged red
   as overdue. Other projects still show if their next application falls within 31
   days.

API
---
/api/applications?upcoming=1 now also returns each draft's appDate, valDate and
monthKey so the table can render dates and detect overdue drafts.
