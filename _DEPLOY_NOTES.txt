Fixes: valuation-day note, project-page vs EOM reconciliation, banner scope
===========================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully

1) "Valuation day: ⚠ Not set" REMOVED
   On the project page it now reads, in light grey:
     • "Valuation day: 28th of each month"  (if a fixed day is set), or
     • "Valuation day: using monthly dates"  (for projects on per-month dates)
   No more amber warning.

2) RUSSELL HILL FIGURES NOW RECONCILE WITH EOM  ← the real cause
   The project page (calcAtDate) was computing Invoiced/Margin on a DIFFERENT basis
   from the EOM report:
     • Project page used invoice TOTAL (inc VAT) then grossed retention UP on top.
     • EOM used account-code-200 Sales (net of VAT, already incl. retention).
   These never matched even at the same date. calcAtDate now uses the SAME sales200
   basis as EOM and the Budget Tracker, so at the same valuation date the Invoiced,
   Gross Invoiced and Margin figures reconcile. (Combined with the earlier valuation-
   date fixes, project page and EOM now line up.)

3) "PROJECT DETAILS INCOMPLETE" BANNER — now scans ALL projects
   The banner was only looking at the currently-FILTERED rows, so projects hidden by
   the stage/date/card filters were missed. It now scans the full project list (minus
   the shared hide list), so every incomplete project is listed regardless of the
   active view.
   NOTE: the banner reads detailsMissing from the dashboard cache. If it's still empty
   after deploying, the cache predates GroupD2 — open the page once (or Sync) to
   rebuild it, and ensure lib/projectComplete.js exists in GitHub.
