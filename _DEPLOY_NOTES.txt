WHAT CHANGED
------------
The Company field on the Add/Edit user form is now an AUTOCOMPLETE:
  • As you type, it suggests existing companies (already on record) that contain
    what you've typed — pick one to use its exact spelling.
  • If what you've typed exactly matches an existing company (ignoring case and
    surrounding spaces), a green "✓ Matches an existing company — this record
    will be used" note appears.
  • On save, if the typed name matches an existing company (case/whitespace-
    insensitive) it is normalised to the existing canonical spelling — so
    "rock roofing", "Rock Roofing" and "Rock Roofing " all collapse to the one
    company already on record instead of creating duplicates.
  • Suggestion list closes when you click outside it.

Belt-and-braces: the SERVER also de-duplicates on both create and edit, so a
duplicate can't slip through even if the client list is stale.

The company list is derived from existing Site App Users (that's where company
lives), so it grows automatically as you add new companies.

FILES CHANGED

• pages/operations/users.js  — CompanyInput autocomplete; client de-dupe on save
• pages/api/ops-users.js     — canonicalCompany() de-dupe on create + edit
