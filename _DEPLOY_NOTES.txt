DEPLOY NOTES - Budgets page crash FIXED (React #310, hooks order)
=================================================================

ROOT CAUSE
----------
React error #310 = "rendered more hooks than during the previous render". The page's
totals useMemo was placed AFTER the early `if (!ok) return null` auth gate. On the
first render (before auth resolves) that hook didn't run; once authed it did - one
more hook than before -> crash. This was my mistake when I added the totals row, and
it breaks the rule in our own notes: ALL hooks must be before any early return. It was
not caused by the sync data - the page would crash regardless; the sync just happened
to be when you reloaded.

THE FIX
-------
Moved the totals useMemo (and the visibleAccounts / completedFyMonths derivations it
uses) ABOVE the `if (!ok) return null`. All hooks now run on every render. The error
boundary added last time stays in (harmless; it will now simply never trigger for this).
