This fix needs BOTH files. If only the page deploys and the API doesn't, the
fresh-email log call hits an old API that doesn't know the 'log-email' action,
and you'll see "Email sent, but logging it failed: Unknown action" in the compose
box. So please make sure BOTH of these land on GitHub:
  • pages/outstanding-invoices.js
  • pages/api/outstanding-invoices.js

HOW TO CONFIRM THE RIGHT BUILD IS LIVE
--------------------------------------

refresh (Ctrl/Cmd+Shift+R) or wait for Vercel to finish, then retry.

---------------------------
Draft fresh email → type Subject + Body → pick/enter a To → Send:

        📧 Email sent to bob@acme.co.uk
        Subject: <subject>
        <full body…>
  • Template sends behave the same (logged with full body).

IF IT STILL DOESN'T LOG
-----------------------
The compose box now shows the exact reason. Tell me the message:
  • "…logging it failed: Unknown action"  → the API file didn't deploy.
  • "…logging it failed: 401/403"         → a session/role issue on that user.
  • "Send failed: …"                       → the SEND failed (Resend/env), so it
                                             correctly did not log.
