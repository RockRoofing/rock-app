PLANNING — Phase 3 refinements: set dates easily + historic view

  pages/operations/project-management/planning.js

  1) SET PROJECT DATES DIRECTLY ON THE ROW
     The Start and "Contract Compl." cells on each project row are now CLICKABLE. Click either one to
     open a focused "Programme dates" dialog with Project start date + Contracted completion date and
     Save. This is the clear way to build the programme — you no longer have to go via a day cell.
     (The day pop-out still also lets you set dates; both write to the same place.)
     Empty date cells now show "⚠ set date" on a pale-red background so they're obvious.

  2) HISTORIC VIEW TOGGLE
     New "Historic" button (top-right, next to Back/Today/Fwd). When on, the Gantt also shows past
     weeks (it pulls the start back by the current zoom span) so you can look backwards. Off by default.

  Note on building the Gantt bars: a project's day cells only show a number once you've assigned named
  installers to that day (click a day cell -> add installers -> Save day). The bar = installers on site.
  Set the project's Start/Completion dates first (click the date cell) so the row is anchored, then
  allocate installers per day.

Deploy over the previous planning.js. (Uses the same Phase 2 API — no API change.)
