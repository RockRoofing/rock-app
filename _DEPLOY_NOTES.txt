★★★ AFTER DEPLOYING — YOU MUST RE-UPLOAD ★★★
The Unassigned-invoices fix changes how invoices are processed ON UPLOAD. Invoices
already in the database were imported by the OLD code (which dropped untagged ones)
— deploying does NOT retroactively fix stored data. So:
  1. Deploy this package.
  2. Go to Admin -> Xero Upload and RE-UPLOAD the Sales Invoices file
     (use SalesInvoices...11_56_03.csv — the recent one; the 11_56_54 file is old
     history and contains NONE of your current outstanding invoices).
  3. The 15 untagged outstanding invoices will then appear on Outstanding Invoices
     with an amber "UNASSIGNED" badge (total ~£84,919). All 28 outstanding invoices
     (~£280,797) will show: 13 under projects + 15 unassigned.
  4. Also re-upload Bills + Direct Wages so the Bookkeeping page has untagged data.
  5. Trigger GET /api/cron/deep-sync once to populate the Bookkeeping "In Xero"
     benchmark immediately (otherwise it fills overnight).

If figures still look stale after re-upload, it's the dashboard cache (4h) — any
upload clears it, or hit /api/clear-cache.
