Bookkeeping — FIX: cost bills appearing under Sales Invoices
============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy, THEN clear + re-sync (steps below).

THE BUG
-------
Sales Invoices tab showed cost/bank items (Fuel card, Bibby, reimbursements) and
"loads of cost bills", across old dates too — bills had polluted the sales data.

THE FIX (two guards)
--------------------
"Type guard" = only accept genuine SALES invoices. In Xero, sales = ACCREC,
bills = ACCPAY. Both entry points now reject anything that isn't a real sales
invoice:
• Sync Invoices (API): skips any record where Type !== 'ACCREC', and skips
  DELETED/VOIDED. So a re-sync can never pull a bill into Sales again.
• CSV invoice upload: explicitly skips rows whose Type contains "bill" (fixes a
  hole where "Bill credit note" slipped through the credit-note check).

CLEARING THE EXISTING POLLUTION (important — old dates too)
----------------------------------------------------------
The guards stop NEW pollution but don't remove what's already stored (you saw
wrong items on Jan 2026 and earlier). So after deploying:

1. Deploy this.
2. Bookkeeping Admin -> Data Management -> Clear Sales Invoices (type CLEAR).
   (Clear scans ALL keys incl. archived projects, so it removes every polluted
   item, old and recent.)
3. Bookkeeping dashboard -> set "Sync last" to the window you want (e.g. 24 mo to
   get older sales back) -> press ↻ Sync Invoices.

Result: Sales will contain ONLY genuine ACCREC sales invoices, matched to
projects by tracking name. Header will show "X matched, Y unassigned (of Z)".

FILES CHANGED
-------------
• pages/api/sync-invoices.js        — ACCREC-only guard + skip deleted/voided
• pages/api/import-invoices-bulk.js — CSV: exclude bills (incl. bill credit notes)
