WHERE
-----
• In the Site App's Contracts Manager area there's a new card: "📄 Applications".

HOW IT BEHAVES
--------------
• PROJECT FIRST: pick a project. Only projects where the logged-in user IS the
  Contracts Manager are shown (useMyProjects — matches the CM field on the project).
• APPLICATIONS TABLE: lists ALL applications (any status) for the project, showing:
    - Application no.
    - Required application date
    - Status (Draft / Sent / Paid)
    - This cert value
  Sorted newest application first.
• TAP AN APPLICATION: opens the full summary with ALL sections and columns:
    - Summary block (required/valuation/payment dates, measured contract sum,
      variations final, anticipated final account, application total)
    - Certificate block (Previously / To date / This application × Gross, MCD,
      Sub-total, Retention, Total)
    - Contract Works (item, description, qty, unit, rate, total, % complete)
    - Variations (VO, description, status, final value, % complete)
    - Materials on Site (description, qty, unit, rate, total, claimed %)

DATA / SAFETY
-------------
• New read-only GET endpoint /api/applications-view (no writes). Added to the Site
  App middleware passthrough. It computes the same figures as the portal via
  computeApplicationSummary / resolveAppDates.
• This is the CM area, so financial columns ARE shown (unlike the operative Schedule
  of Works). Access is still gated to the user's own CM projects.
