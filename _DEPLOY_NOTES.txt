DEPLOY NOTES - "Average Time to Get Paid" still empty: robust fix + diagnostics
===============================================================================

The first fix added fullyPaidOnDate to the sync, but the card can still be empty
for TWO other reasons. This update handles all three and adds an on-card diagnostic
so we can SEE exactly which one it is instead of guessing.

THE THREE CAUSES
----------------
1. IN-PROGRESS ONLY (big one): the metric was only looking at invoice lines on
   IN-PROGRESS projects. But invoices are usually fully paid AFTER a job has moved
   to defects/closed - so most paid invoices were being excluded. FIXED: payment
   times now scan ALL projects (still excluding the hidden list). The Gross Profit
   figures are unchanged and still in-progress only.

2. TOO-STRICT PAID GATE: it required status === 'PAID' exactly. Now an invoice
   counts as paid if status is 'PAID' OR it shows nothing outstanding
   (amountDue == 0 and amountPaid > 0). Credit notes still excluded.

3. FIELD NOT SYNCED: fullyPaidOnDate only exists on lines written AFTER the earlier
   fix. Still needs a re-sync if the last sync predates it. (Note: wip-sync already
   wrote this field, so depending on which sync ran last it may already be present.)

WHAT WAS ADDED TO DIAGNOSE IT
-----------------------------
The API now returns paymentDiag counters, and the card shows a small amber line
WHEN (and only when) there are zero qualifying paid invoices, e.g.:

  "No qualifying paid invoices. Of 812 invoice lines: 0 have a paid-on date,
   140 read as paid. -> Re-sync invoices to capture paid-on dates."

That tells us immediately whether it's the sync (0 have a paid-on date) or the
gates. Once real data shows, the amber line disappears automatically.

HOW TO DEPLOY
-------------
Replace both files in the GitHub web editor. Commit -> Vercel auto-deploys.

THEN
----
1. Open the Commercial Scorecard.
2. If the card now shows days - done.
3. If still empty, READ THE AMBER LINE on the card:
   - "0 have a paid-on date"  -> re-sync invoices (Sync bar on a Commercial page),
     then reload. If wip-sync is what runs on your schedule the field may already be
     present after the next scheduled run.
   - "N have a paid-on date" but 0 qualify -> tell me the numbers from the amber
     line and I'll adjust the gate; that means the dates are there but something
     else is filtering them (e.g. all outside the 0-365 day sanity window).
