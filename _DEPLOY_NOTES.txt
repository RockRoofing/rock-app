CHANGE
------
Added an "On Hold" status to App Improvements:
  • New "Put on hold" button on each open item (and "Take off hold" to return it to
    Open). Putting on hold sends NO email to the reporter.
  • On Hold items are HIDDEN from the default "Open" view (the default filter is Open,
    which now excludes both Resolved and On Hold).
  • New "On Hold" option in the Status filter to view them, and a grey "On Hold"
    label on those cards.
  • Statuses are now: Open / On Hold / Resolved.

No API change needed (the endpoint stores the status as-is and only emails on
transition to Resolved).
