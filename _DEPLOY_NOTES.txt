Site App Users — fixes + RAMS Matrix freeze/wider column
========================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of the previous fixes.

1) COMPANY + TRADE NOT SAVING ON CREATE  (main bug)
---------------------------------------------------
When adding a NEW user, the server built the record from an explicit field list
that omitted company + trades, so they were dropped (edit worked because it
spreads all fields). Fixed: company + trades are now saved on create, so the
Site App Users table populates correctly first time — no re-edit needed.

2) PIN NO LONGER HAS TO BE UNIQUE  (security fix)
-------------------------------------------------
Login is by MOBILE NUMBER + PIN, so two users may share a PIN as long as their
mobiles differ. Removed the "that PIN is already in use" block on the user's
set-PIN step, and simplified temp-PIN generation (no longer avoids collisions).
Mobile numbers remain unique (they identify the user at login).

3) WELCOME EMAIL WORDING
------------------------
• Subject: "Welcome to the Rock Roofing Site App" (was "…Forms")
• Body references the Site App, and the button now reads "Open Rock Roofing
  Site App" (was "Open Rock Roofing Forms"). PIN-reset subject updated too.

4) ADD-USER FORM
----------------
• Every field now has a red * (First/Last name, Role, Access level, Project
  access, Mobile, Email, Company) and Trade(s) shows "* (select at least one)".
  All are already enforced on save (name, role, mobile, valid email, company,
  ≥1 trade).
• Project-access dropdown now CLOSES when you click anywhere outside it.
• Fixed the cramped spacing: the helper text under Project access now sits with
  proper spacing (was too close).

5) RAMS MATRIX — wider project column + frozen header & first column
-------------------------------------------------------------------
• Project column widened (220→300px) and the stage line no longer wraps, so
  "… › Operatives" stays on one line.
• The grid now scrolls in BOTH directions inside a fixed-height viewport, with:
    – the operative-name HEADER row frozen to the top on vertical scroll, and
    – the "Project RAMS" column frozen to the left on horizontal scroll
  (the top-left corner stays pinned above both).
