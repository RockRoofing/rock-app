ADMIN — Documents manager (Company Information + Operative Guidance Docs)

pages/admin/documents.js  (NEW)
  - New "Documents" page in the Admin centre. Two sections: Company Information and Operative Guidance
    Documents.
  - Per section: upload a file, give it a NAME, click Upload -> it appears as a card. Cards can be renamed
    inline and deleted. "View" opens the document.
  - Uploads go through the existing /api/upload-file (Vercel Blob) and are saved into ops:docs via
    /api/ops-docs — the SAME store the Site App reads, so a doc added here appears immediately as a named
    card in the Site App under Company Information / Operative Guidance Docs.

components/AdminShell.js
  - Added a "Documents" tab to the Admin nav (between Site App Users and Problem Reports).

Deploy both files. No cron/env change. (Uses existing Blob upload + ops:docs storage — nothing new to set up.)
