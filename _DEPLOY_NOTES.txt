Financial — one-time "Reset Direct Wages data" (clear old lump sums)
====================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy over the top.

WHY
---
Old untagged wage LUMP SUMS are already stored (from the long 2023–2026 file).
The new correctly-tagged wages file would sit ALONGSIDE them (dedupe can't match a
lump sum to its per-project split), so wages would double-count. You wanted a
clean reset — but NORMAL uploads must still merge and never wipe.

WHAT THIS ADDS
--------------
A "Reset Direct Wages data" panel (red danger zone) at the bottom of
Admin -> Xero Upload. Admin-only, two-step confirm. When run it:
  • Deletes every per-project wage source (costs:wages:<id>)
  • Deletes the untagged wage lump sums (costs:untagged:wages)
  • Re-runs mergeCosts per project so project cost totals drop the wages
  • Clears the dashboard cache
  • Does NOT touch bills or invoices
The API requires an explicit confirm flag, so it can't fire by accident.

NORMAL UPLOADS ARE UNCHANGED — they still merge/dedupe and never wipe. Only this
button clears data.

USAGE
-----
1. Deploy.
2. Admin -> Xero Upload -> "Reset Direct Wages data" -> confirm.
3. Upload the CORRECT tagged Direct Wages export (the one WITH a "Projects"
   column — 27 columns). Your April file (Account_Transactions__3_) is the right
   format: 17 projects, ~£195k tagged + £12,300 genuinely untagged.
4. Repeat step 3 for any other periods/batches you want tagged.
