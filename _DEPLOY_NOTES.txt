1) MANUAL ENTRIES ARE STANDALONE (never touch Project Financials)
-----------------------------------------------------------------
Anything added via "Add Manual Entry" now lives ONLY in the Retention Tracker.
• Using the Project Name autocomplete to pull an existing project's details just
  COPIES those values into the entry as a snapshot — it does NOT link the entry
  to the project. So a manual entry never writes back to Project Financials and
  never appears there.
• The two-way COMMENTS sync now only applies to rows that come from Project
  Financials automatically (genuine Xero-sourced rows). Manual entries — even if
  you auto-filled them from a project — do not sync comments anywhere.

2) NO DUPLICATE PROJECTS
------------------------
The autocomplete no longer suggests a project that's already in the tracker —
whether it's there as an auto/linked row or as an existing manual entry (matched
on xeroId, ref or project name). So you can only add projects that aren't already
listed.

FILE CHANGED
------------
• pages/retention.js — manual entries flagged trackerOnly and never linked
  (no xeroId), autocomplete filters out projects already in the tracker.
