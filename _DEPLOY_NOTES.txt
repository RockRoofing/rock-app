DEPLOY NOTES - Overhead Budgets: cell drill-down, colour key, forecast-driven budget,
 4 summary columns, Dec/Jan fix
=====================================================================================

CHANGES
-------
1) CLICK A MONTH CELL -> transaction pop-out (date, description, reference, amount,
   with a total that reconciles to the cell). Data is the GENERAL LEDGER captured at
   sync time (includes bills, bank payments AND manual journals - so it sums to the
   P&L figure, unlike bills-only). Stored per account/month; clicking reads from
   Redis instantly (no per-click Xero calls).
   - Pulled by the existing "Sync Xero figures" button (once-daily is fine). It full-
     replaces the last window each sync, so back-dated bills, credit notes, edits and
     voids are picked up on the next sync.

2) COLOUR SCHEME + KEY (completed months vs monthly budget):
     BLACK  = within 15% under budget
     ORANGE = more than 15% under budget
     RED    = over budget
   A key is shown above the table. Future months stay grey (forecast).

3) FORECAST DRIVES THE BUDGET/MO COLUMN. If you haven't typed a budget, the Budget/mo
   field shows the forecast figure (amber text) as the suggested budget; type to set
   your own. Changing the forecast method updates the suggestion.

4) "Same as last month" forecast option is now "3 - Last full month" and uses the
   most recent COMPLETED month's actual for that account.

5) FOUR SUMMARY COLUMNS after the last month:
     - Budget to date   = monthly budget x completed months this FY
     - Actual to date   = sum of completed months this FY
     - Full-yr budget   = monthly budget x 12
     - Tracking (yr)    = projected full year (actual to date + forecast for the
                          remaining months) minus full-yr budget. Red if over,
                          green if under.

6) DEC / JAN SHOWING BLANK: this was because the benchmark hadn't been re-synced
   since it was extended to 13 months - Dec 2025 / Jan 2026 weren't in the stored
   data. After deploying, click "Sync Xero figures" once; the 13-month pull now
   includes them. (If a month is still blank afterwards it genuinely has no data in
   Xero for that period.)

API-LIMIT NOTE
--------------
The ledger is pulled ONCE per sync (paging the Journals endpoint), not per cell click.
Cell clicks read stored data. Xero limit is 60/min, 5000/day; a sync uses ~35-55 calls
total. Once daily is comfortable.
