DEPLOY NOTES - Bills page: hard guard against sales invoices (ACCREC)
====================================================================

WHY IT WAS STILL MIXING
-----------------------
The bills feed requests Xero Type=ACCPAY (supplier bills), so in theory sales invoices
(ACCREC) can't appear. Since you still see them, either (a) the STORED bills data was
saved by older code and contains sales invoices, or (b) Xero is returning mixed types
for that query. I can't see your Redis/Xero from here, so this fix handles BOTH and
proves which it is.

WHAT THIS DOES
--------------
1) SERVER (lib/xero.js): the invoice fetch now HARD-FILTERS by inv.Type in code (not
   just via the query param) and stamps each record with its `type`. So a re-sync can
   never store an ACCREC record in the bills key.

2) PAGE (bills.js): a client-side GUARD drops any record whose type is ACCREC before
   anything is displayed - so even the CURRENT stored data is shown bills-only without
   needing a sync. Records with no type (older data from the bills feed) are treated as
   bills unless explicitly ACCREC.

3) DIAGNOSTIC BANNER: if any non-bill records are found in the stored data, a red banner
   shows how many were hidden. This tells us whether sales invoices really were baked
   into storage. Clicking "Sync bills" then refreshes from Xero and clears them.

WHAT I NEED FROM YOU AFTER DEPLOY
---------------------------------
- If the red banner appears ("Hidden N records that were NOT supplier bills"): the
  stored data had sales invoices. Click "Sync bills" once - the banner should then go

- If NO banner appears but you STILL see something that looks like a sales invoice:
