Chart of Accounts sync + Overheads/Uncategorised categorisation
===============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
NEW FILE to create in GitHub: pages/api/sync-chart-of-accounts.js

1) SYNC CHART OF ACCOUNTS  (Account Categorisation admin area)
   • New "↻ Sync chart of accounts" button pulls EVERY account from Xero (code +
     name, via the Accounts endpoint) into the list. Shows last-synced time.
   • New API: /api/sync-chart-of-accounts (POST, admin/accounts/management only).
   • Stored at config:chart-of-accounts; the categorisation list merges it in, so
     all account codes + names appear.

2) "IGNORE" RENAMED TO "OVERHEADS"
   The third category is now "Overheads" (was "Ignore"). Overheads are excluded from
   project costs, same as before. Legacy 'ignore' values are auto-migrated to
   'overheads' everywhere (display, classification, save).

3) NEW ACCOUNTS DEFAULT TO UNCATEGORISED (flagged)
   • Any account code with no category assigned is "Uncategorised" — EXCLUDED from
     project costs (like Overheads) and flagged. Shown with a red row + "needs
     categorising" in the admin list, and counted in the summary.
   • Unknown codes seen in cost data also default to Uncategorised (previously they
     silently defaulted to Ignore).

4) BOOKKEEPING APP BANNER
   The Bookkeeping app shows a red banner listing every uncategorised account code
   (code + name) with a link to the Account Categorisation page. It appears whenever
   any code in the data hasn't been categorised.

AFTER DEPLOY
------------
1. Open Admin → Account Categorisation and click "Sync chart of accounts".
2. Assign any Uncategorised codes to Labour / Materials / Overheads and Save.
3. Re-sync/re-upload bills so costs recalculate with the new categories.
