Project Financials mirrors the WIP page (WIP margin read-only) + override sync
==============================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: Compiled successfully
Deploy AFTER the WIP_shared_calc_sync bundle.

CHANGES
-------
1. PROJECT FINANCIALS IS NOW A MIRROR — the WIP Margin column on Project Financials is
   READ-ONLY. You can no longer edit/override the WIP margin there; it just displays
   the effective margin ("*" = overridden on the WIP page). Set WIP margin overrides
   on the WIP page only. PF reflects them.

2. OVERRIDE NOW PICKED UP — two reasons it wasn't:
   • The dashboard dropped a 0% override (used "|| null"); now preserved with a
     proper nullish check, so any value incl. 0% flows through.
   • Manual WIP adjustment changes now also clear the dashboard cache, so PF refreshes
     to match the WIP page (margin overrides already cleared the cache; adjustments
     didn't).

WHY IT LOOKED STUCK
-------------------
PF reads WIP from the dashboard cache (4h). Setting an override on the WIP page busts
that cache, but adjustment edits previously didn't — so PF could lag. Now every
WIP-affecting change (override OR adjustment) refreshes PF on next load.

FILES
-----
• pages/commercial.js                        (WIP margin column read-only)
• pages/api/dashboard.js                      (preserve 0% override)
• pages/api/project/[id]/wip-adjustments.js   (bust dashboard cache on change)

AFTER DEPLOY
------------
Set/clear a WIP margin override on the WIP page, then open Project Financials — it
should now show the same WIP and margin. (First load rebuilds the cache.)
