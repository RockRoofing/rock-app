WHAT'S NEW IN THE COMPOSE POPUP
-------------------------------
1. CC CONTROLS
   • A CC field with removable chips — add/remove anyone (incl. customer contacts).
   • Auto-seeded from the template's flags: if the template is set to CC the Rock
     Roofing CM, the CM's email is pre-added (resolved by matching the project's
     Contracts Manager NAME to a team member's email).
   • Quick-add chips: "+ Rock CM (name)" and "+ Customer contact" when those
     emails are known.
   • If a template normally CCs the CUSTOMER SITE MANAGER, a note prompts you to
     add their email (there is no dedicated site-manager field on projects yet —
     see LIMITATION below).

2. DRAFT FRESH EMAIL
   • Toggle at the top: "[Stage] template" vs "Draft fresh email".
   • Fresh mode clears subject/body for a fully custom message and is NOT tied to
     the timeline (sending it does not tick a stage).

3. MANUAL TIMELINE ADJUSTMENT
   • In the popup, a row of stage checkboxes lets you tick/untick any stage by
     hand — e.g. if you chased by phone or sent a custom email, mark the stage
     done without sending. Uses new 'record-chase' (manual:true) / 'clear-chase'
     actions. The dot turns green just like a real send.

CC DATA SOURCES (dashboard.js now exposes these per project)
   customerEmail, customerContact  (from project settings → Customer Contact)
   contractsManager (name) -> matched to team member email for the Rock CM CC.

LIMITATION / FUTURE
   There is no dedicated "customer site manager" field on projects, so that
   particular auto-CC can't pre-fill — the user adds it manually (prompted). If
   you want it automatic, add a customerSiteManagerEmail field to project
   settings and we'll wire it in. Same for the remaining blank merge fields
   (Customer Address, Customer QS Name/Email, Project Address, Sub-Contract Ref)
   — tell me where to store them and they'll auto-resolve.
