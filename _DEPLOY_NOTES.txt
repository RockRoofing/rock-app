Fix: Schedule of Works showed no projects for operatives
========================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Single file: pages/forms/schedule.js

CAUSE
-----
The Schedule of Works page used useMyProjects().myProjects, which is filtered to
projects where the user is the CONTRACTS MANAGER. Operatives aren't CMs, so that list
was empty — hence "no projects".

FIX
---
It now uses useMyProjects().projects — the list filtered by the operative's own
project PERMISSIONS (projectAccess). So an operative sees exactly the projects they're
allowed to view (all of them if projectAccess is 'all'/unset, otherwise just their
assigned ones), matching the spec.

FILES
-----
• pages/forms/schedule.js

NOTE: if a specific operative still sees nothing, check their projectAccess in the
Site App user setup — an empty/explicit list with no projects would correctly show
none. 'all' or unset shows every project.
