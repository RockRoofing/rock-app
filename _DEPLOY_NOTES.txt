THE BUG
-------
Sync Invoices ran across all 26 projects but returned 0 invoices, so Sales
"Categorised in app" stayed £0 — even though invoices ARE project-tagged in Xero
(confirmed from your June export: INV-1695 -> J242-Winnersh, etc.).

Cause: the API path matched invoices to projects by tracking-option GUID
(line.Tracking[].TrackingOptionID === trackingOptionId). That comparison was
returning nothing. Meanwhile the CSV path matches by NAME ("J242-Winnersh") and
works. Two different mechanisms for the same goal — the GUID one was broken.

THE FIX
-------
Sync Invoices now matches by tracking-option NAME — the same reliable text match
the CSV upload uses:
  • ONE pass: fetch all ACCREC (sales) invoices, newest-first, stopping once past
    the window (default 6 months) — far fewer API calls than the old per-project
    scan.
  • For each invoice, read its line-item tracking OPTION label (e.g.
    "J242-Winnersh") and match to the project by name.
  • Group per project; unmatched -> Unassigned. Exact-mirror within the window
    (older invoices preserved).
Result message now shows: "X matched to projects, Y unassigned (of Z)".
