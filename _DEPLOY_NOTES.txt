Disable nightly deep-sync cron (move to manual syncing)
=======================================================
Deploy target: RockRoofing/rock-app  branch main

WHY
---
The overnight deep-sync was the only unattended process writing to every
project's cost keys, and it's the main drift/risk. Manual syncs (transaction
upload, wages sync, invoice sync, "Sync Xero figures") all work and are fast, so
we're switching to manual-only.

CHANGE
------
vercel.json: removed the deep-sync cron entry. The endpoint /api/cron/deep-sync
STILL EXISTS and can be triggered manually if ever needed — it's just no longer
scheduled. The other two crons (deliveries-sync, hs-expiry-email) are unchanged.

WHAT NOW UPDATES MANUALLY (all confirmed covered)
-------------------------------------------------
• Project bills      -> manual Bills export/import (import-bills-bulk)
• Untagged/OVERHEADS -> SAME bills import: untagged lines are routed to
                        costs:untagged:bills automatically. Export all bills for
                        a date range and overheads are included. (verified)
• Wages              -> manual Sync Wages
• Sales invoices     -> manual invoice sync/upload
• Grey P&L figures   -> "↻ Sync Xero figures" button

NOTE (Vercel Hobby/Pro cron limits)
-----------------------------------
Fewer crons is always safe. If a future vercel.json change ever stops deploys,
suspect a cron-count/schedule limit — not relevant here since we're REMOVING one.

FILE CHANGED

• vercel.json
