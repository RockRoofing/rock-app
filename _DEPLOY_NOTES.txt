DEPLOY NOTES - Fix: "Average Time to Get Paid" showing no data
==============================================================

WHY IT WAS BLANK (root cause)
-----------------------------
The metric needs three fields on each stored invoice line: date, status ('PAID'),
and fullyPaidOnDate. commercial-metrics.js filters:

    inv.fullyPaidOnDate && inv.date && inv.status === 'PAID'

BUT the main invoice sync (pages/api/sync-invoices.js) that writes invoiced:lines
NEVER captured fullyPaidOnDate - the line object had date, dueDate, status,
amountPaid, amountDue, but no fullyPaidOnDate. So inv.fullyPaidOnDate was always
undefined, the filter matched zero invoices, and the card showed nothing.

(Note: pages/api/cron/wip-sync.js already maps this field correctly as
`fullyPaidOnDate: inv.FullyPaidOnDate`, which confirms the correct Xero field name
is FullyPaidOnDate. The main sync-invoices.js was just missing it.)

THE FIX
-------
1. pages/api/sync-invoices.js
   - Sales invoice lines now capture: fullyPaidOnDate: full.FullyPaidOnDate || null
   - Credit-note lines intentionally do NOT get this field (a credit note is not
     "getting paid").

2. pages/api/commercial-metrics.js
   - paidInvoices filter now also excludes credit notes explicitly
     (&& !inv.creditNote) so they can never skew the average.

FILES (both EDITS, no new files)
--------------------------------
  - pages/api/sync-invoices.js
  - pages/api/commercial-metrics.js

HOW TO DEPLOY (GitHub web editor)
---------------------------------
Replace both files at their paths. Commit -> Vercel auto-deploys.

*** IMPORTANT - YOU MUST RE-SYNC AFTER DEPLOYING ***
The fix only adds the field going forward. Existing stored invoice lines were
written WITHOUT fullyPaidOnDate, so the card stays blank until you re-sync:
  1. Deploy both files.
  2. Go to a Commercial page with the Sync bar (e.g. Project Financials) and run
     the invoice sync so invoiced:lines are rewritten WITH fullyPaidOnDate.
  3. Open the Commercial Scorecard - "Average Time to Get Paid" should now show
     days, with a trend and a clickable drill listing the paid invoices.

IF IT'S STILL BLANK AFTER A FULL RE-SYNC
----------------------------------------
That would mean Xero's Invoices *list* endpoint isn't returning FullyPaidOnDate
for your org (it usually does). In that case the next step is to source the field
the way wip-sync.js does. Tell me and I'll wire that in - but a normal re-sync
should populate it.
