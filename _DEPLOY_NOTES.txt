Deploy, then re-run Sync Wages.

WHAT WAS WRONG
--------------
Wages now pull in line-by-line (great!) but with NO date, so they can't be
filtered or assessed by month.

Cause: the ManualJournals LIST response often omits BOTH the line detail AND the
header date. The code took the date from the list entry (null) and stamped that
null onto every line. When it re-fetched the journal by ID for the lines, it
ignored the fuller detail's date.

THE FIX
-------
When fetching a journal by ID (for lines/tracking), also read the header date from
that detail response and use it. The by-ID detail reliably carries Date/DateString.
So every wage line now gets its correct journal date. Also triggers the by-ID
fetch when the list entry lacks a date (rate-limit safe).

AFTER DEPLOYING
---------------
Re-run Sync Wages. Wage lines should now show their dates and be filterable by
month, and land in the correct month on the Direct Wages card/graph.
