Application PDF — each variation's documents follow its own description
======================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Single file: lib/applicationPdf.js

WHAT WAS WRONG
--------------
Image attachments already sat under their variation header, but PDF attachments were
deferred to a single block at the very END of the document. So if a variation's
supporting document was a PDF, it ended up separated from its variation — unclear
which docs belonged to which variation.

THE FIX
-------
In the "Variation Supporting Documents" section, every variation's documents now come
directly after THAT variation's header/description:
  • Images: embedded inline beneath the header (scaled to fit, capped ~70% page
    height), as before.
  • PDFs: their pages are now embedded INLINE immediately after the header too (with
    a small caption on the first page), instead of being appended at the end.
So the order is: VO — description — total, then that variation's documents, then the
next variation. Material/supplier documents follow at the end of the section.
