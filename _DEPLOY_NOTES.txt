PART 1 — Project Details reflects calendar-entered dates
--------------------------------------------------------
Dates entered via the Application Calendar (fixed days AND manual monthly
overrides) already persist to the same project settings, so they show in Project
Details. Change: the monthly override table now OPENS BY DEFAULT (pre-filled with
the existing dates) whenever the project already has any manual dates — so the
user immediately sees what the dates are based on, rather than a collapsed/blank
table.
  [pages/project/[id].js — showDateOverrides defaults open when dateOverrides exist]

PART 2 — EOM report date resolution (bug fix)
---------------------------------------------
The EOM report resolved each project's valuation date from the FIXED valuation
day only, and returned nothing if no fixed day was set — so projects with only a
MANUAL monthly date dropped out of EOM, and a manual override never took
precedence over the fixed day.
Fix (pages/commercial.js — calcAtValDate): same precedence as the calendar —
  1) manual per-month override (dateOverrides[month].valuationDate) wins;
  2) else the fixed valuationDay (clamped to month length);
  3) if neither, no valuation date for that month.
So EOM now works whether dates are fixed-per-month OR entered manually per month.
