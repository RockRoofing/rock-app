FRESH ("Draft fresh email") FIX
-------------------------------
The fresh send path uses the same /api/chase-email-send as templates, so it does
send — but two things made it look broken:
  1) No up-front validation: a blank subject/body only failed at the server
     ("Subject is required"), which looked like nothing happened.
  2) The fresh branch closed the modal WITHOUT logging anything, so a successful
     send left no trace on the invoice.
Now:
  • Send validates recipient, SUBJECT and BODY before sending, with a clear
    message in the compose box if any are missing.
  • A fresh send is LOGGED as a comment (new 'log-email' API action), so you can
    see it on the invoice just like template sends.
  • Any real send/logging error is shown in the compose box (not swallowed).

FULL EMAIL TEXT IN COMMENTS (both template + fresh)
---------------------------------------------------
Sent emails logged to the invoice comments now include the FULL email:
      📧 Overdue 1 email sent to bob@acme.co.uk
      Subject: Overdue invoice INV-1234
      <the entire body that was sent…>
The comment view renders with pre-wrap, so the body reads exactly as sent. The
raw fields are also stored on the comment (comment.email = { to, subject, body })
for future use. These flow into the weekly report as before.
