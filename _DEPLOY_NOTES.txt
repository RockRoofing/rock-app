RAMS — FIXES: chain emails, Director signing, matrix pipeline, RAMS badges, CM email
====================================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of Phase 4 (these files supersede those copies).

1) DIRECTOR EMAIL + FULL CHAIN HANDOFF EMAILS (was: Director got no email)
-------------------------------------------------------------------------
Emails now fire as the chain advances:
  • RAMS uploaded  -> emails the project's CONTRACTS MANAGER to approve (chain
    starts with them; previously it wrongly emailed all operatives).
  • CM approves    -> emails the designated DIRECTOR to approve/sign in the Site App.
  • Director approves -> emails the CM to send it to the Site Manager.
  • Site Manager approves -> emails the OPERATIVES that they can now sign.
The CM is matched to a Site App user by the project's Contracts Manager name.
The Director is the Admin → RAMS Director (matched by email OR name).

2) DIRECTOR SIGNING IN THE SITE APP (was: only "View", no approve button)
-------------------------------------------------------------------------
The Site App now recognises the Director by email OR by name (so it still works
if their Site App login email differs from their Portal email). When they open a
RAMS at the Director stage they get "✓ Approve as Director" (read-to-bottom →
sign). Server accepts the approval on email OR name match too.
NOTE: you must still set Admin → RAMS Director, and that person must be a Site
App user. If nothing appears, check the name/email matches on both records.

3) RAMS MATRIX PIPELINE (was: CM red after signing, Operatives green wrongly,
   Director grey while with him)
----------------------------------------------------------------------------
Root cause: CM & Director AUTO-SIGN onto the RAMS when they approve, and those
auto-signatures were being counted as "operative" signatures — turning the
Operatives node green and skewing the line. Fixed: the matrix now EXCLUDES the
CM/Director auto-signatures (tagged by role / cm:/director: ids) from the
operative signer set. The pipeline now reflects the true stage:
  completed stages green, current stage red+bold, Operatives node greens only
  when a real operative signs, whole line green when all shown have signed.
(Use the ↻ Refresh button to pull the latest state.)

4) RED RAMS BADGES ON INDIVIDUAL PROJECTS (was: only the home total)
-------------------------------------------------------------------
The RAMS section's project picker now shows a red count on each project that has
RAMS awaiting THIS operative's signature. Badges (home + per-project) now only
count RAMS that have reached the operatives stage AND are unsigned — so nobody
gets a red badge for RAMS still going through CM/Director/Site-Manager approval.

5) CM ALLOCATION EMAIL WORDING
------------------------------
When the person allocated is the project's Contracts Manager, the email now
reads "Rock Roofing project allocated" and states they're allocated as
Contracts Manager (not the generic operative wording).
