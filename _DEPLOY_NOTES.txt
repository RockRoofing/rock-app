PLANNING — PHASE 3 of 5: the Gantt UI

  pages/operations/project-management/planning.js   Replaces the coming-soon stub. The Gantt:
   - Rows: LIVE PROJECTS first, then NEGOTIATED — NOT YET SECURED (clearly labelled, amber tint).
   - Columns: day view starting Monday; W/C date header per week; day letters M T W T F S S with the
     date; weekends shaded; Monday has a heavier divider.
   - Each project-day cell shows the installer COUNT. Click a cell to open the day pop-out.
   - Top row tallies TOTAL installers per day across all visible projects.
   - Start + Contracted Completion date columns beside the project name. ⚠ shown if either date is
     missing (in the date cell and by the project name). The contracted-completion day has a red right
     edge; any allocation PAST the completion date is shaded red, and a ⚠ over-run warning shows by the
     project name.
   - Filters: Project, Installer (shows only projects that installer is on), From/To week range.
   - Navigation: Back / Today / Fwd, and zoom by weeks shown (4/8/12/26).

  DAY POP-OUT (click any cell):
   - Set the project Start + Contracted Completion dates here (writes to planning meta).
   - Add named installers from the Operatives roster; set Full day / AM / PM per person; remove.
   - The picker flags installers already "busy" elsewhere that day. On save, the server clash-checks:
     a person can't be on two projects the same day unless BOTH are half-days (AM on one, PM on the
     other) — a clash returns a clear error naming the person.
