WHY THE PREVIOUS RUN WAS SLOW / TIMED OUT
-----------------------------------------
The date-fix forced a fetch-by-ID for EVERY journal across ALL history, at ~400ms
each -> minutes, hitting the Vercel function timeout (partial saves).

WHY WE CAN'T USE THE FASTER /Journals ENDPOINT
----------------------------------------------
The /Journals endpoint needs scope accounting.transactions.read. Under Xero's NEW
granular scopes (this app is on them), that broad scope no longer exists — the
app's allowed list only has granular ones (banktransactions.read, manualjournals.
read). So /Journals is not accessible. ManualJournals (accounting.manualjournals.
read, which we now have) is the correct endpoint.

THE SPEED FIX
-------------
Query ManualJournals with a date-filtered `where` clause:
    Date>=DateTime(y,m,d)
so we only pull journals in the selected window instead of paging through all
history. Per-item fetch-by-ID now only happens for the rare journal missing
lines/date, and only over that small window set. Tighter, rate-limit-safe sleeps.
