1) CRs POPULATE EDIT-PROJECT-DETAILS BUDGETS ON LOCK
   Locking CRs overwrites ONLY the contract baseline:
     • Original Contract Value  <- CR "Above the line" total
     • Labour budget            <- CR labour (above)
     • Materials budget         <- CR materials (above)
   Variations you've added are stored separately and still stack ON TOP (total
   budget / AFA = baseline + instructed variations). Nothing variation-related is
   touched.

2) PROJECT FINANCIALS NOW UPDATE IMMEDIATELY  ← the fix you just hit
   Budget Tracker + EOM read a 4-hour dashboard cache. Locking CRs now CLEARS that
   cache, so the next time you open Project Financials it rebuilds from the updated
   project details and matches. (First load after a lock is a little slower as it
   rebuilds from live Xero data — expected.)

3) SEARCHABLE PROJECT DROPDOWN ON THE CRs TAB
   Type-to-search by job no or name.

4) SAVE → PROMPT TO LOCK
   "Lock Contracted Rates" already saves AND locks in one action. After a plain Save
   you now get: "Are you ready to Lock Contract Rates? This must be done to populate
   the project details and make applications available." Confirm = save+lock.

5) STICKY SECTION TOTALS
   The Above/Below the line totals stick to the bottom of the view and update live as
   you move lines across the line.

6) HEADING CHECKBOXES
   Headings now have a tick box, so a heading can be selected and moved above/below
   the line along with items.
