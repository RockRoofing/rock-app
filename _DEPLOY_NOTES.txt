DEPLOY NOTES - Sales: fix empty table (income-code mismatch) + two dashed lines
===============================================================================

WHY THE TABLE WAS EMPTY (the real reason)
-----------------------------------------
The chart is built from the P&L benchmark, which keys sales by the account NAME mapped
to a code. Your sales P&L figure may sit under a code that ISN'T 200 (e.g. a different
income account). The sales LEDGER pull was only fetching code 200 (+ codes tagged
'sales'), so if your sales journals use a different income code, the ledger had nothing
- hence an empty table even though the chart had bars.

FIX: both the sync (ledger pull) and the sales view now treat ANY code the P&L puts in
the INCOME section as a sales code - not just 200. So the ledger now pulls the same
codes the chart is built from, and the table fills.

You MUST RE-SYNC after deploying so the ledger is re-pulled with the broader code set:
  Deploy -> hard-refresh -> click "Sync Xero figures" once.

DIAGNOSTIC (so we can confirm)
------------------------------
The sales API now returns a `diag` object. If the table is still empty after a sync,
open (while logged in) /api/business-financials?view=sales in the browser and send me
the "diag" section - it lists:
  - benchmarkSalesCodes: the codes (and totals) the chart is built from
  - ledgerCodesPresent: the codes the sales ledger actually holds
If these two lists differ, we'll see exactly which income code needs including.

TWO DASHED LINES (target + average)
-----------------------------------
When a target is set there are now clearly TWO lines:
  - GREY dashed = the target (draws as soon as you type it, before/after saving).
  - GREEN or RED dashed = the average revenue for the shown period; GREEN if the
    average is at/above target, RED if below.
Both are thicker now and set to extend the chart's scale so they can't be hidden off
the top. The key above the chart labels all three (Target / Average / Now).
