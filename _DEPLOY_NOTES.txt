====================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy on top of the batch 3a retention files.

WHAT CHANGED
------------
When manually adding / editing a retention entry, the QS field is now a DROPDOWN
of portal users who have access rights of:
  • Standard — Post-Contract
  • Management
  • Admin
(pre-contract users are excluded.)

• The list is pulled live from Portal Users (Admin → Portal Users) and filtered
  by their access role. Only active users appear.
• If a row already has a QS name that isn't in the current list (e.g. a legacy
  value or someone whose access changed), it's preserved as an option so it isn't
  lost.

FILES CHANGED
-------------
• pages/api/team.js   — expose each user's accessRole (post-contract/management/admin)
• pages/retention.js  — QS field is now a dropdown filtered to those access roles
