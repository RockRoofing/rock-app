Add Sales category + exclude balance-sheet accounts (e.g. 612 Retention)
========================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Deploy AFTER the previous chart-of-accounts / reconciliation bundles.

1) NEW "SALES" CATEGORY
   • Categories are now: Labour · Materials · Overheads · Sales · Uncategorised.
   • Sales is its OWN category — a revenue code is tagged Sales; it's never
     labour/materials/overheads and never a project cost.
   • The Sync now pulls REVENUE accounts too (as well as expense), so sales/income
     accounts appear in the list to be tagged. Code 200 defaults to Sales.
   • The Bookkeeping reconciliation's Sales figure now sums ALL codes tagged Sales
     (not just hardcoded 200), so new sales types reconcile correctly.

2) BALANCE-SHEET ACCOUNTS EXCLUDED  (fixes 612 Retention showing up)
   • The Sync only brings in P&L accounts (Class EXPENSE or REVENUE). Balance-sheet
     accounts (ASSET/LIABILITY/EQUITY) are excluded.
   • More importantly: once a chart has been synced, the categorisation list is
     restricted to synced (P&L) codes + anything you've explicitly saved. Codes that
     previously leaked in via cost-data "seen" (like 612 Retention appearing on
     invoice lines) are no longer added. 612 is also hard-excluded as a known
     balance-sheet code.

AFTER DEPLOY
------------
1. Admin → Account Categorisation → "Sync chart of accounts" (re-pull; now incl.
   revenue, excl. balance sheet). 612 and other balance-sheet codes will drop off.
2. Tag your revenue account(s) as Sales; assign any Uncategorised codes; Save.
3. "Sync Xero figures" then reconcile as before.
