Management role — access to a restricted Admin area
===================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully

WHAT CHANGED
------------
Management users now get the SAME "Admin" button on the portal home and enter the
Admin area, but with a RESTRICTED set of tabs.

Management CAN access:
  • Site App Users
  • Documents
  • RAMS Director
  • App Improvements
  • Templates
  • Form Builder

Management CANNOT access:
  • Portal Users (admin only)
  • Bookkeeping tools (admin/accounts as before)

HOW IT WORKS
------------
• pages/index.js — the "Admin" button now shows for management as well as admin.
  For admin it lands on /admin (Portal Users); for management it lands on
  /operations/users (their first permitted tab), since they can't open Portal Users.
• components/AdminShell.js — each admin tab is tagged with the roles allowed to see
  it; the sub-nav now hides tabs the current role can't use. Portal Users shows for
  admin only; the other six show for management + admin.
• Each of the six pages passes allow={['management','admin']} to the shell (or, for
  Templates, widened its inline guard). Portal Users stays admin-only.
• APIs: templates.js and rams-director.js widened to ['management','admin']. The
  others (forms, ops-docs, report-problem, blob-upload) weren't role-gated.
