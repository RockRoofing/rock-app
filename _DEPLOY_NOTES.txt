Bookkeeping — FIX: Sync Invoices only pulling ~2 months instead of the full window
==================================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: npm run build -> ✓ Compiled successfully
Deploy, then re-run Sync Invoices.

THE BUG
-------
A 6-month sync only returned ~2 months. Cause: the fetch ordered invoices
"newest-first" and STOPPED paging as soon as it hit a page with nothing in the
window. But Xero's sort order is unreliable — so it hit a page that looked
out-of-window and bailed early, missing older in-window invoices.

THE FIX
-------
Now uses Xero's server-side WHERE filter so the API only returns sales invoices
dated on/after the window start:
  where = Type=="ACCREC" AND Date>=DateTime(YYYY,M,D)
and pages through ALL results (no fragile early-exit). So the full selected window
comes back (6/12/24 months as chosen).

AFTER DEPLOYING
---------------
Re-run ↻ Sync Invoices with your chosen window. It'll now pull the whole period.
(If you want more than 6 months, pick 12 or 24 in the "Sync last" selector.)

FILES CHANGED
-------------
• pages/api/sync-invoices.js — server-side date filter, removed early-exit
