WHAT THIS IS
------------
The first of three phases for the Outstanding Invoices chase-email feature. This
phase only adds the 5 standard email templates and a place to edit them. Nothing
sends yet, and the Auto Send column still shows "— soon —". Phases 2 (timeline +
compose/send popup) and 3 (CC controls + fresh/custom email + manual timeline
adjust) follow.

                                       config:chase-email-templates, merged over
                                       the code defaults.
• pages/commercial-email-templates.js — editor page at /commercial-email-templates

THE 5 TEMPLATES
---------------
upcoming, overdue1, overdue2, overdue3, withdrawal — worded as supplied. The
withdrawal letter now uses [Customer Company Name] (the "Torsion Construction
Limited" line was replaced). overdue1/2/3 + withdrawal default to auto-CC the
customer site manager and the Rock Roofing CM (toggleable per template).

MERGE FIELDS (typed literally, replaced at send in Phase 2)
[Customer First Name] [Customer Company Name] [Customer Address] [Customer QS Name]
[Customer QS Email] [Invoice Number] [Invoice Reference] [Project Name]
[Project Address] [Sub-Contract Ref] [Due Date] [Today's Date] [Invoice Value]
[Invoice Value inc VAT] [Rock Roofing QS Name]

*** PREREQUISITE for Phase 2 sending (do this when you can) ***
Emails will send via Resend FROM accountsreceivable@rockroofing.co.uk with the
project QS as reply-to. For that to work, rockroofing.co.uk must be VERIFIED in
Resend: add the SPF/DKIM/return-path DNS records Resend provides for the domain.
Until verified, Resend only sends from its test address. You do NOT need to
create an accountsreceivable@ mailbox to SEND (reply-to is the QS); create one
only if you want a home for replies/bounces.
