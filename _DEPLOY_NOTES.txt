NEW BOOKKEEPING CARD + PAGE  (/bookkeeping)
-------------------------------------------
Purple "Bookkeeping" card on the portal home (visible to Accounts/Management/Admin).
The page lists transactions that have NO project tracking category in Xero — i.e.
costs/sales/wages that aren't attributed to a project — so you can reconcile the
app against Xero each month and fix tags at source.

• THREE TABS: Costs (Bills) | Sales Invoices | Direct Wages.
• FILTERS: month, supplier/customer, and account (cost) code.
• RECONCILIATION SUMMARY at the top (per selected month or all):
   - In Xero (cost of sale / sales)  — from the P&L benchmark (see below)
   - Categorised in app              — what's attributed to projects
   - Difference (Xero − app)         — green "reconciles ✓" or red "does not match"
   - Untagged in this view           — the leakage total for the current filter
   - Missing account codes           — cost-of-sale codes seen in Xero data but NOT
     set up in Admin -> Account Categorisation (so you can add them). Untagged
     lines whose code isn't in the app also show a red "NOT IN APP" badge.

HOW THE DATA GETS THERE
-----------------------
• The Bills and Direct Wages importers now CAPTURE untagged lines (previously they
  were silently skipped) into costs:untagged:bills / costs:untagged:wages
  (merged/deduped). Untagged sales invoices already flow via the __UNASSIGNED__
  bucket. Real-file test: 132 untagged bill lines (£54,102), including cost-of-sale
  codes 311 & 330 that had no project tag — exactly the leakage to catch.
• XERO BENCHMARK: a Profit & Loss pull (lib/xero.js fetchProfitAndLoss) gives the
  month's account totals in ONE API call per month. Folded into the nightly
  deep-sync (current + previous 3 months, once/day) and cached at xero:pl-benchmark
  — so it does NOT hammer the Xero API and needs no extra cron slot.

NOTES
-----
• The "In Xero" and "Difference" figures appear after the first overnight deep-sync
  runs (or trigger GET /api/cron/deep-sync once to populate immediately).
• Sales vs cost split in the P&L is inferred by account name (Sales/Income/Revenue
  -> sales; else cost). If your chart of accounts uses unusual names we can refine
  the classification.
