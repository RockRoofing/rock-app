1) PC DATE / DEFECTS DATE "TBC" OPTION (edit project details)
   Each of the two retention dates now has a "TBC" checkbox next to it. Tick it to
   mark the date as intentionally to-be-confirmed:
     • the date input is cleared/disabled while TBC is ticked
     • TBC counts as "complete" for the details-complete check (below)
     • the Retention tracker shows "⚠ TBC" on that release date, with a tooltip
       noting it was marked TBC in project details.
   Stored as pcDateTBC / defectsDateTBC on the project.

2) "PROJECT DETAILS NOT COMPLETE" BANNER (Project Financials + Retention)
   A thin amber banner (same style as the Application Calendar "Missing dates" bar)
   appears above both pages listing every project whose edit-details are incomplete.
   Each project is an underlined link that goes straight to its Project Details page.
   Hovering a link shows exactly which fields are missing.
   Required fields (all must be filled): Application day, Valuation day, Payment day,
   Contract value, Labour budget, Materials budget, Retention %, PC date (or TBC),
   Defects date (or TBC). Defined once in lib/projectComplete.js and used by both
   pages (via the dashboard's detailsMissing field).
