FIX: can't set users to Pre-Contract / Post-Contract

ROOT CAUSE: the portal user save API (portal-auth.js) still validated roles against the OLD
list ['standard','management','admin']. So:
  - Management saved fine (it was in the old list)
  - Pre-Contract / Post-Contract were rejected: on CREATE they fell back to 'standard';
    on UPDATE the role change was DELETED (discarded) — which is why editing to Pre/Post-Contract
    "didn't take".

FIX (pages/api/portal-auth.js):
  - Both create and update now validate against the central ROLES list
    (pre-contract, post-contract, management, admin), with legacy 'standard' normalised
    to post-contract.

Deploy over the previous portal-auth.js. After deploy: edit a user -> set Pre-Contract ->
Save -> reopen; it should now stick. (This is the real fix; the earlier admin.js change was
only the dropdown display.)
