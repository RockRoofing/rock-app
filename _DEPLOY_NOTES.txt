NEW PAGE: "Outstanding Invoices"  (/outstanding-invoices)
--------------------------------------------------------
Sits in the commercial nav right AFTER Project Financials (link added to the
Project Financials, Retention, Variations and Application Calendar nav bars).
Aesthetic matches the existing portal pages.

Columns (mirrors your Xero list in principle):
  • Inv Number  — invoice number (+ red "HIGH RISK" badge if the customer is flagged)
  • Ref         — the invoice reference (e.g. "Farmstead Drive - App 3")
  • To          — customer name
  • Date        — invoice date
  • Due Date    — turns RED once passed
  • Overdue by  — days overdue in red; blank when not overdue
  • Expected date — manual date entry (saved per invoice)
  • Paid        — amount paid
  • Due         — balance outstanding
  • Comments    — opens a pop-out (see below)
  • Auto send emails — placeholder ("— soon —"); the chase-email buttons + timeline
                       come in a later batch once you provide the email content.

Only invoices with a balance still due are listed. Data comes from the invoice
cache built by the Xero CSV import / sync — NO extra Xero API calls.
Summary cards: outstanding count, total due, overdue count, overdue value.
Search + sort (most overdue / largest due / due date).

COMMENTS POP-OUT (per invoice)
------------------------------
• A running chain of submitted comments, each with author + date/time.
• Add new comments (the chain grows); existing comments are retained.
• @mention portal users by typing "@" — a picker appears; mentioned users get an
  in-app notification ("<author> mentioned you on invoice <number>").
• Comments added via the future email-BCC route will show a "via email" tag
  (plumbing is in place; the BCC intake itself comes in a later batch).

HIGH-RISK CUSTOMER  (item 5)
----------------------------
Edit Project Details → under Customer name there is now a tick box
"Is this a high risk customer?". When ticked, that customer's invoices show a red
"HIGH RISK" badge on the Outstanding Invoices page.
