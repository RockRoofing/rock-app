In the monthly application/valuation/payment date-override table, each date box is
now constrained to its own month via min/max (first & last day of that row's
month). So the native date picker OPENS on the correct month — e.g. clicking a
November box opens November, not today's month — and only that month's dates can
be entered.

Includes the earlier changes to this file (Costs-tab filters, and the table
showing the last 2 + 12 months ahead). This is the latest full version of the file.
