FIX: SRAT & Project Concern task/risk linking wasn't persisting

ROOT CAUSE: "+ Add Task" / "+ Add Risk" created the row with an EMPTY description, but the
tasks/risks APIs REJECT creation when description is empty (400). So the new task/risk never
actually saved to Live Project Tasks / the Risk Log — it looked half-wired.

FIX:
  pages/api/tasks.js            require description only on CREATE (new), allow updates with any value
                                 (so inline-editing to empty mid-type doesn't fail).
  components/ProjectConcerns.js  Add Action / Add Risk now create with a placeholder ("New action" /
                                 "New risk") so they persist immediately; edit to rename. Alerts on failure.
  pages/operations/project-management/srat.js  Add Task creates with "New task" placeholder; alerts on failure.

RESULT: adding an action/risk in a Concern or SRAT now immediately creates the linked record on
the Live Project Tasks / Risk Log pages (Concern = two-way, SRAT = one-way), as intended.
