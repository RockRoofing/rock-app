Applications — variations behaviour rework
==========================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully  ·  Applications tab shows "· v4"
Deploy all three files (on top of Phase 3).

WHAT CHANGED
------------
1) ALL PROJECT VARIATIONS APPEAR AUTOMATICALLY on the application (no more "add
   from tracker" step). Drafts read them LIVE from the Variation Tracker.

2) STATUS DRIVES TOTALLING:
   • Instructed  → normal black text, included in totals, has % Complete (with the
     ✓=100% button) and a Value to date.
   • Not instructed → the whole line is greyed, there is NO % Complete input, and
     Value to date shows "N/A". It does NOT contribute to the totals or the
     Summary/Certificate.

3) MARK INSTRUCTED FROM THE APPLICATION: click the status pill on a variation to
   toggle Instructed / Not instructed. This writes back to the Variation Tracker
   (settings.variations), so Project Details, budgets and WIP all update from the
   same source. (Instructed variations feed those figures already.)

4) SENT = FROZEN, DOUBLE-CLICK TO EDIT: when an application is marked as sent, its
   variations are FROZEN as they are at that moment. A sent application is locked;
   "Double-click here to edit this sent application" unlocks it for changes (same
   pattern as the Contracted Rates lock), then Save re-persists.

DATA MODEL
----------
• Drafts store only per-variation data (variationData keyed by varNumber+desc):
  the % complete and any attachments. The variation identity/value/instructed
  come live from the tracker.
• On "Mark as sent" the full computed variation list is frozen into the
  application (application.variations) so the sent document never changes if the
  tracker later does.
• New API action: set-variation-instructed (matched write-through to the tracker).
