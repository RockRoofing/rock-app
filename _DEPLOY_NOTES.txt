Commercial — hide projects across the commercial views (shared setting)
=======================================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully

WHAT
----
A settings-icon (⚙) dropdown on Project Financials (next to the project count)
lets any logged-in user tick/untick which projects are visible. The list is SHARED
(stored server-side) so one user's change is seen by everyone. Default = visible;
a newly-added project shows everywhere until someone hides it.

Hiding a project removes it from:
  • Project Financials (Budget Tracker + EOM)
  • Retention
  • Application Calendar
  • Commercial Scorecard (metrics recomputed without hidden projects)

Use it to hide non-project categories that aren't relevant (Internal, Stock Metal,
Overheads, etc.).

HOW IT WORKS
------------
• pages/api/hidden-projects.js (NEW) — GET/POST a shared array of hidden project
  ids at config:hidden-projects.
• components/HideProjectsDropdown.js (NEW) — the ⚙ dropdown: search, tick/untick,
  Show all / Hide all, Save. Ticked = visible.
• commercial.js — icon added by the project count; baseFiltered excludes hidden.
• retention.js — excludes hidden from the list, picker and merged entries.
• application-calendar.js — excludes hidden in-progress projects.
• commercial-metrics.js — scorecard metrics exclude hidden projects.

The edit UI lives only on Project Financials; the other pages just honour the list.
