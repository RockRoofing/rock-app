WIP page — banner, default month, per-adjustment margin
=======================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
Deploy AFTER the WIP_page and WIP_page_fixes bundles.

CHANGES
-------
1. MISSING-DATES BANNER — same amber banner as the Applications / Application Calendar
   pages, listing projects that don't have application + valuation dates set. Clicking
   a project opens the EXACT same ProjectDatesModal pop-out (monthly option + per-month
   override table) used on those pages. Saving refreshes the WIP page.

2. DEFAULT MONTH — the WIP page now defaults to the LAST COMPLETED month (previous
   month) instead of the current month. You can still filter back through earlier
   months.

3. INFO-ONLY ADJUSTMENTS now show the full project name (job no — name) for each
   last-month adjustment.

4. PER-ADJUSTMENT MARGIN — each manual WIP adjustment now has its own margin %, which
   DEFAULTS to the project's current margin. Each adjustment's WIP contribution is
   grossed up at its own margin; the project WIP = post-valuation costs at the project
   margin + each adjustment at its own margin. Existing adjustments show their margin
   (or "project" when they inherit it).

FILES
-----
• pages/api/wip.js                              (missingDates list; per-adjustment margin calc)
• pages/wip.js                                  (banner + modal, default last month, name in info banner, margin field)
• pages/api/project/[id]/wip-adjustments.js     (store per-adjustment margin)

NOTE: the missing-dates banner reads the dashboard cache, so load Project Financials
once if the list looks empty on a cold cache.
