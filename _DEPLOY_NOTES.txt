NAV
---
• "WIP" tab added to the shared Commercial nav, BEFORE Commercial Scorecard. Because
  the nav is shared, WIP shows on every commercial page, and all the other tabs show
  on the WIP page.

THE PAGE
--------
• Total WIP shown at the top.
• "Figures as at valuation date" month filter at the top — defaults to the CURRENT
  month, and you can filter back through previous months. Because it defaults to the
  current month, on the 1st the next WIP month shows automatically: manual
  adjustments (which are stored per-month) drop off the sections, and the new costs
  from valuation date → end of month appear.
• Last month's manual adjustments show in an amber banner at the top FOR INFORMATION
  ONLY — they are NOT added to the total (so you can see if those known WIP items
  should still apply).
• One section per project, headed with the project name + job no and the valuation
  date for the selected month.
• Under each project, two columns side by side:
   - LEFT (wider): costs from the DAY AFTER the valuation date to end of month (the
     same table as the project WIP tab: date, supplier, invoice, account, type,
     amount, with a total). If the valuation date IS the end of month, nothing is
     listed.
   - RIGHT (narrower): credit notes issued against the project — date, credit note
     number, the invoice it was applied to, and the amount.
   - Below the left column: the Manual WIP Cost Adjustments editor (add/remove,
     positive = increases WIP, negative = write-down). Adjustments are per-month.
• WIP is totalled per project (in each project header) and overall (top).

DATA
----
Reads costs:lines:{id}, invoiced:lines:{id} (credit notes flagged), and
wip:adjustments:{id} per project, plus each project's settings for valuation date and
margin. Manual adjustments use the existing /api/project/{id}/wip-adjustments endpoint.

Access: post-contract, management, admin (same as other commercial pages).
