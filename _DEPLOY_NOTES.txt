Fix: Schedule of Works showing no projects (stale-cache flag)
=============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully

CAUSE
-----
The Schedule filtered the project list on p.hasContractedRates. That flag is new, so
a dashboard cache built before it existed returned it as undefined on every project —
filtering them ALL out -> empty picker. (The cache guard only forced a rebuild when
detailsMissing was missing, not this new flag.)

FIXES
-----
1) dashboard.js: the cache guard now also requires hasContractedRates, so a cache
   lacking it is ignored and rebuilt automatically.
2) schedule.js: made resilient — if NO project in the returned list carries the
   hasContractedRates flag (stale cache), it no longer filters on it; it shows all
   accessible projects and the CR check happens when one is opened ("No schedule
   uploaded" if none). Once the cache rebuilds, it filters to CR projects as intended.

RESULT
------
Projects appear in the Schedule picker again. Permissions (projectAccess) still apply.
