Materials on Site — line rates, global mark-up, % claimed
=========================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully  ·  Applications tab shows "· v6"
(The "Attempted import error" lines during build are pre-existing warnings from
other files, unrelated to this change — the build still compiles successfully.)

1) LINE ITEM RATES NOW PULL IN
   Root cause: PO line items were fetched from Xero with description + quantity
   ONLY (no cost), so no rate reached the application. lib/xero.js now also pulls
   UnitAmount (rate) and Unit for each PO line.
   IMPORTANT: existing ops:deliveries rows were synced before this change and have
   no rate stored yet — run a SYNC on the delivery schedule to refresh them (new
   and re-synced POs will then carry rates). Until then, older rows show a blank
   rate you can type in manually.

2) MARK-UP ALL MATERIALS
   The Materials on Site header has a "Mark-up all __%" box — type once and it sets
   the mark-up on EVERY material line. Per-line mark-up still editable, and the
   per-PO mark-up in the picker still works.

3) % CLAIMED FOR MATERIALS ON SITE
   New "% Claimed" column (with a ✓=100% button), exactly like Contract Works.
   Value to date = marked-up Total x % claimed. New lines default to 100%.
   • The TOTAL row now shows both the marked-up final total and the value-to-date.
   • Summary/Certificate "Materials On Site" now uses the VALUE TO DATE (what's
     being certified this application), and the carry-forward accounts for it.
