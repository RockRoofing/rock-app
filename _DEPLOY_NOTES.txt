WHAT WENT WRONG
---------------
Sync Wages deleted your wage data. Two faults combined:
1. ROOT CAUSE: the ManualJournals LIST endpoint returns journals WITHOUT their
   line detail (JournalLines/Tracking) — so the fetch found no 320 lines and
   returned EMPTY.
2. The store loop then cleared in-window wages for every project and replaced
   them with nothing (empty) = wiped.

THE FIXES
---------
1. fetchAllWageLines now fetches each journal BY ID when the list omits its lines
   (429-safe), so it actually gets the 320 tagged lines. This is why it was empty.
2. SAFETY GUARDS so a bad/empty pull can NEVER wipe data again:
   • If the whole fetch returns 0 lines -> abort, change nothing, report a note.
   • A project is only overwritten if the fetch returned lines FOR THAT PROJECT;
     otherwise its existing data is left untouched.
   • The untagged bucket is only replaced if untagged lines were actually fetched.

TO RECOVER YOUR WAGES
---------------------
1. Deploy this.
2. Re-run ↻ Sync Wages. It should now fetch the journal lines properly and
   repopulate ("X tagged to projects, Y unassigned (of Z)").
3. If any period still looks short, re-upload that period's correct wages export
   (27-col, with Projects column) as the fallback.
