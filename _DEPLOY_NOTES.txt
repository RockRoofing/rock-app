WHAT CHANGED
------------
The missing-dates banner on the Applications page now opens the EXACT SAME box the
Application Calendar uses — extracted into a shared component so they can never
drift apart:
  • Amber "⚠ Missing one or more dates…" notice
  • "Fixed monthly dates" — Application / Valuation / Payment day tiles
  • "＋ Add monthly dates manually" — the per-month override table (Month /
    Application / Valuation / Payment date pickers, 14-month window)
  • "Save dates" + "View project ↗"
Saving writes through to the project settings (applicationDay / valuationDay /
paymentDay / dateOverrides) exactly as the calendar does, so Project Details, the
calendar and the upcoming table all pick it up. On the Applications page the
upcoming table + banner refresh immediately after saving.

The Application Calendar now renders the same shared component for its per-project
modal (its "all projects on this day" overflow popup is unchanged).
