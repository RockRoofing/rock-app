Site App — "Schedule of Works" for operatives (no financials)
=============================================================
Deploy target: RockRoofing/rock-app  branch main
Build: ✓ Compiled successfully
NEW FILE to create in GitHub: pages/forms/schedule.js

WHAT IT IS
----------
A read-only Contracted Rates view for operatives inside the Site App, with NO
financial information.

WHERE
-----
• In the operative home menu (the Operative Permissions area) there's a new card:
  "📐 Schedule of Works". Tap it to open.

HOW IT BEHAVES
--------------
• PROJECT FIRST: you pick a project. It uses the SAME permission model as the rest of
  the Site App (useMyProjects) — an operative only sees the projects they have
  permission to view. If you can only see project X, only X's schedule is available.
• SCHEDULE ITEMS: shows ONLY item number, description, quantity and unit. No rates,
  no line totals, no section totals — none of the financial columns are rendered.
• VARIATIONS: listed at the bottom with VO number, description, item and quantity
  (where present) and an "Instructed / Not instructed" badge. No values, no
  materials/labour/profit — no financial info at all.

DATA / SAFETY
-------------
• Reuses the existing read-only /api/contracted-rates-view endpoint (GET only). That
  endpoint now also returns variations with NON-FINANCIAL fields only (varNumber,
  description, item, qty, unit, instructed) — the financial fields are never sent to
  the Site App for this view.
• Routing already covered by middleware (/forms/* and /api/contracted-rates-view are
  in the Site App passthrough) — no middleware change needed.
